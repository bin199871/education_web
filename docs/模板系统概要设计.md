## 1. 系统架构

### 1.1 总体架构

```
┌─────────────────────────────────────────────────────────┐
│                    前端 (index.html)                      │
│  用户粘贴题目 → 点击"生成" → 新标签页打开独立HTML         │
└────────────────────────┬────────────────────────────────┘
                         │ POST /api/template/generate
                         ▼
┌─────────────────────────────────────────────────────────┐
│                   后端 (FastAPI + Python)                  │
│                                                          │
│  ┌──────────────┐  ┌──────────────┐  ┌───────────────┐  │
│  │  Layer1      │  │  Template    │  │  LLM          │  │
│  │  题目分析    │→│  引擎        │→│  参数提取     │  │
│  └──────────────┘  └──────┬───────┘  └───────────────┘  │
│                           │                               │
│                    ┌──────┴───────┐                       │
│                    │  模板文件     │                       │
│                    │  (HTML文件)   │                       │
│                    │  ① 读取      │                       │
│                    │  ② 注入参数   │                       │
│                    │  ③ 返回HTML   │                       │
│                    └──────────────┘                       │
└─────────────────────────────────────────────────────────┘
                         │ 返回完整HTML
                         ▼
┌─────────────────────────────────────────────────────────┐
│                 浏览器新标签页                             │
│   physics-pendulum.html 级别的自包含讲解动画               │
│   播放/暂停/进度条/速度切换/快捷键                         │
└─────────────────────────────────────────────────────────┘
```

### 1.2 四层职责

| 层 | 模块 | 职责 |
|----|------|------|
| 触发层 | `index.html` / API调用 | 用户输入题目，触发生成流程 |
| 分析层 | `Layer1 (layer1_agent.py)` | 分析题目主题、核心概念、标签 |
| 匹配层 | `TemplateEngine (template_engine.py)` **新增** | 匹配模板 + LLM提取参数 + 注入渲染 |
| 呈现层 | 模板 HTML 文件 | 自包含的完整讲解动画 |

## 2. 模块设计

### 2.1 TemplateEngine 模块

```python
class TemplateEngine:
    """
    模板引擎：模板注册 → 匹配 → 参数提取 → 渲染
    """

    # ----- 注册表 -----
    REGISTRY: dict  # 模板ID → {file, label, params, tags}

    # ----- 核心方法 -----
    list_templates()           → list[dict]
    match_template(analysis)   → str | None       # 标签评分制
    extract_params(problem, template_id) → dict   # LLM提取
    render(template_id, params) → str             # 注入→输出HTML
    validate_params(template_id, params) → bool   # 参数校验

    # ----- 内部方法 -----
    _build_extraction_prompt(problem, tpl) → str
    _call_llm(prompt) → dict
    _read_template(template_id) → str
    _inject_params(html, params) → str
```

### 2.3 模板公共基座

所有模板共享的公共代码——帧控制器、场景调度器、UI控制器等可以提取为公共模块。

**方案一：内联复用（推荐初版）**
每个模板保留完整代码，通过复制+修改的方式创建新模板。简单直接，适合模板数量不多时。

**方案二：公共 JS 库（推荐后期）**
将帧控制器、场景调度器、UI控制器提取为独立的 `.js` 文件，模板通过 `<script src>` 引用。

**初版采用方案一**，因为：
- 每个模板保持完全自包含，没有外部依赖
- 模板间互不影响，修改一个模板不会影响其他
- 方便分享——单个 HTML 文件就是完整动画

## 3. 核心流程设计

### 3.1 模板匹配算法

```
输入: Layer1 分析结果 { topic, concepts, tags, ... }

步骤:
  1. 为每个注册模板初始化分数 = 0
  2. 标签精确匹配:  topic/tags 中包含模板 tag → +10
  3. 关键词模糊匹配: 题目文本中包含模板 tag 关键词 → +5
  4. 主题相关性:     topic 与模板 label 相似度 → +3 (使用简单关键词)
  5. 取最高分模板
  6. 最高分 > 0 → 返回模板ID
  7. 最高分 = 0 → 返回 None (降级)

示例:
  题目: "在竖直平面内存在水平向右的匀强电场..."
  topic: "电场"
  tags: ["电场", "圆周运动"]
  
  评分:
  - electric_pendulum: 标签"电场"+"复合场" → +20 + 关键词"匀强电场"→+5 = 25
  - inclined_plane: 无匹配标签 → 0
  - projectile: 无匹配标签 → 0
  
  结果: electric_pendulum (得分 25)
```

### 3.2 LLM 参数提取流程

```
输入: problem_text, template_id

步骤:
  1. 从 REGISTRY 获取模板的 param_schema
  2. 构建 extraction_prompt:
     - 模板标签 + param_schema + 示例输出
  3. 调用 LLM (通过现有 httpx/urllib 接口)
  4. 解析 LLM 返回的 JSON
  5. 校验参数是否符合 schema (类型/范围)
  6. 缺失参数 → 用 schema.default 填充
  7. 注入题目文本/问题/答案到 params
  8. 返回完整 params dict

LLM Prompt 模板:
  """你是一个高中物理题参数提取器。
  
  题目文本：
  {problem}
  
  题型：{template_label}
  
  请从题目中提取以下参数，输出为纯JSON格式（不要多余文字）：
  {param_schema_json}
  
  参数说明：
  {param_descriptions}
  
  规则：
  - 数值用 float，角度用度（°），不要带单位字符
  - 缺失参数用默认值
  - 科学计数法如 5e-4
  - 额外提取题目中的 question1/2/3 和 answer1/2/3
  """
```

### 3.3 模板渲染流程

```
输入: template_id, params dict

步骤:
  1. 读取模板 HTML 文件
  2. 验证 params 完整性（缺失→默认值）
  3. 注入 params → 替换 TEMPLATE_PARAMS 占位
  4. 注入标题 → 替换 [[TITLE]] 占位
  5. 返回完整的 HTML 字符串

注入方式: 字符串替换
  html.replace('var TEMPLATE_PARAMS = null;', 
               f'var TEMPLATE_PARAMS = {json.dumps(params)};')
  html.replace('[[TITLE]]', params.get('title', '物理讲解动画'))
```

## 4. API 设计

### 4.1 新增端点

#### POST `/api/template/generate`

从题目文本生成模板动画。

**请求：**
```json
{
  "text": "在竖直平面内存在水平向右的匀强电场...",
  "template_id": null
  // template_id 为 null 时自动匹配，指定时强制使用
}
```

**响应：**
```json
{
  "success": true,
  "template_id": "electric_pendulum",
  "params": { "m": 0.1, "q": 5e-4, ... },
  "html": "<!DOCTYPE html>..."
}
```

**失败降级：**
```json
{
  "success": false,
  "fallback": true,
  "message": "未匹配到模板，使用标准流水线",
  "redirect": "/api/analyze/storyboard"
}
```

#### GET `/api/template/list`

返回可用模板列表。

**响应：**
```json
{
  "templates": [
    {
      "id": "electric_pendulum",
      "label": "电场中带电单摆",
      "params": ["m", "q", "E", "L", "g", "theta_target"],
      "tags": ["电场", "单摆", "复合场"]
    }
  ]
}
```

#### POST `/api/template/preview`

直接使用指定参数渲染模板（调试用）。

**请求：**
```json
{
  "template_id": "electric_pendulum",
  "params": { "m": 0.1, "q": 5e-4, "E": 2000 }
}
```

**响应：** `text/html` 完整动画页面

### 4.2 server.py 集成

```python
# 在 server.py 中新增

from template_engine import TemplateEngine

template_engine = TemplateEngine()

@app.post('/api/template/generate')
async def template_generate(req: TemplateGenerateRequest):
    # 1. Layer1 分析
    analysis = layer1.analyze_problem_safe(req.text)

    # 2. 匹配模板
    tid = req.template_id or template_engine.match_template(analysis)

    if not tid:
        # 降级到旧流水线
        return await fallback_to_storyboard(req.text)

    # 3. LLM 提取参数
    params = template_engine.extract_params(req.text, tid)

    # 4. 渲染
    html = template_engine.render(tid, params)

    # 5. 写入文件 + 返回
    output_path = 'frontend/premium-output.html'
    with open(output_path, 'w', encoding='utf-8') as f:
        f.write(html)

    return {
        'success': True,
        'template_id': tid,
        'params': params,
        'html': html
    }
```

## 6. physics-pendulum.html → electric_pendulum.html 改造方案

### 6.1 改造清单

| 改动项 | 当前 (physics-pendulum.html) | 改造后 (electric_pendulum.html) |

| 参数区 | 硬编码 `PHYS` 常量 | `TEMPLATE_PARAMS` 注入 + `PARAMS` 合并 |

## 7. 扩展性设计

### 7.1 添加新模板的步骤

```
1. 定义 param_schema → 更新 REGISTRY
2. 复制参考模板 → 重命名
3. 修改物理引擎 → 新题型的计算逻辑
4. 修改场景定义 → 新题型的12场景分镜
5. 修改 Canvas 绘制 → 新题型的视觉元素
6. 修改面板内容 → 新题型的解题步骤
7. 测试独立运行 → 用默认值播放
8. 注册到 REGISTRY → 加入 template_engine.py
```

### 7.2 模板注册表可外部配置（未来）

```json
// templates/registry.json（可选外部配置文件）
{
  "electric_pendulum": {
    "file": "electric_pendulum.html",
    "label": "电场中带电单摆",
    "params": ["m","q","E","L","g","theta_target"],
    "tags": ["电场","单摆","复合场"]
  }
}
```

这样新增模板只需：
1. 把 `.html` 文件放入 `templates/`
2. 在 `registry.json` 中注册
3. 无需修改 Python 代码

**初版暂不实现外部配置，模板注册写死在 REGISTRY 中。** 后续可按需抽取。

## 10. 技术风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| LLM 参数提取不准确 | 动画物理量错误 | 加入 schema 校验，缺失用默认值，输出警告 |
| 模板匹配错误 | 用错题型模板 | 允许前端手动指定 template_id |
| 模板文件变大 | 加载慢 | 保持 50KB 以内，必要时拆分公共库 |
| 新题型无法用现有模板结构表达 | 需要大幅修改框架 | 首批2-3个验证框架通用性，暴露问题早修改 |
| 题目文本中未给出足够参数 | 动画物理量不全 | LLM 用标准默认值补全（g=10, m=1等） |
