## 1. 概述

### 1.1 项目目标

构建一个能够根据自然语言描述的物理题目，自动生成准确、可交互、具有教学解释性的 **Three.js 3D 教学动画** 的系统。

### 1.2 核心原则

```
┌─────────────────────────────────────────────────────┐
│                    物理准确                          │  ← 不可妥协的红线
├─────────────────────────────────────────────────────┤
│                    教学有效性                        │  ← 核心追求
├─────────────────────────────────────────────────────┤
│                交互体验与泛化能力                     │  ← 差异化优势
├─────────────────────────────────────────────────────┤
│              成本可控 | 持续进化                     │  ← 系统生命力
├─────────────────────────────────────────────────────┤
│              可解释性 | 让"为什么"可见               │  ← 教学差异化核心
└─────────────────────────────────────────────────────┘
```

### 1.3 与现有 2D 系统的关系

| 维度 | 2D 系统（现有） | 3D 系统（新建） |
|------|----------------|----------------|
| 渲染技术 | Canvas 2D | Three.js (WebGL) |
| 模板格式 | 独立 HTML + Canvas JS | 独立 HTML + Three.js（基于场景原语组合） |
| 后端管道 | layer1→layer2→layer3→layer4 | 全新 Multi-Agent 系统 + 物理规则抉择引擎 |
| API 端点 | `/api/*` | `/api/3d/*` |
| 前端路径 | `frontend/2d/` | `frontend/3d/` |
| 后端路径 | `backend/` 各模块 | `backend/3d/` |
| 进化能力 | 手动更新规则 | 补丁晋升机制（越用越聪明） |
| 用户分层 | 无 | 新手/进阶/竞赛 三层 |

**两者并行，互不干扰**，用户可根据需要选择 2D 或 3D 渲染模式。

## 4. Multi-Agent 系统设计

### 4.1 Agent 角色定义

#### 4.1.1 主 Agent（Orchestrator）

| 属性 | 说明 |
|------|------|
| **文件** | `backend/3d/agents/main_agent.py` |
| **职责** | 接收题目 → 计算置信度 → 决策走 Agent 路径还是规则直出 → 分发任务 → 汇总 → 处理重试 |
| **输入** | 题目文本 + 用户分层等级 |
| **输出** | `SceneRequirements` |

**工作流程：**

```
1. 接收题目文本 + 用户分层等级
2. 规则预分析：提取关键词、检测题型
3. 计算置信度: score = α×K + β×P + γ×T
   (K=关键词匹配率, P=参数完整性, T=题型匹配度; α=0.3, β=0.4, γ=0.3)
4. 决策：
   - score ≥ 60% AND 模板匹配 → 启动完整 Multi-Agent 流程
   - score < 60% → 规则直出路径（如有匹配模板）
   - 无模板匹配 → 文字解析模式
5. 分发任务给物理专家 Agent 和视觉设计 Agent（并行）
6. 汇总结果 → 发给验证官 Agent
7. 根据验证反馈：
   - 通过 → 输出 SceneRequirements
   - 不通过（第1-2轮）→ 仅重试相关 Agent，增量合并
   - 不通过（第3轮）→ 降级
8. 记录规则使用情况到补丁池
```

#### 4.1.2 物理专家 Agent

| 属性 | 说明 |
|------|------|
| **文件** | `backend/3d/agents/physics_agent.py` |
| **职责** | 识别物理定律、提取参数、从规则抉择引擎选择适用规则、生成运动方程 |
| **输入** | 题目文本 + 题型提示 |
| **输出** | `PhysicsSpec` |

**输出数据结构：**

```python
@dataclass
class PhysicsSpec:
    problem_type: str                    # 题型 ID
    user_level: str                      # 用户分层等级: beginner | advanced | competition
    known_params: dict[str, float]       # 已知参数
    unknown_params: list[str]            # 待求量
    laws: list[str]                      # 适用物理定律
    selected_rules: list[str]            # 从规则抉择引擎选取的规则ID列表
    equations: list[str]                 # 关键方程
    key_values: dict[str, float]         # 计算结果
    motion_equations: list[dict]         # 运动方程（用于动画驱动）
    phases: list[dict]                   # 物理过程阶段
    special_cases: list[str]             # 特殊条件说明
    contradiction_warnings: list[str]    # 矛盾检测警告（如触发）
```

**置信度计算：**

```python
def calculate_confidence(text: str, physics_spec: PhysicsSpec) -> float:
    """
    置信度 = α × 关键词匹配率 + β × 参数完整性 + γ × 题型匹配度
    α = 0.3, β = 0.4, γ = 0.3
    """
    keyword_rate = match_keywords(text)          # 0.0 ~ 1.0
    param_rate = check_param_completeness()      # 0.0 ~ 1.0
    type_rate = check_type_match()               # 0.0 ~ 1.0
    return 0.3 * keyword_rate + 0.4 * param_rate + 0.3 * type_rate
```

| 属性 | 说明 |

### 4.3 Agent 重试策略

```
验证失败时，重试规则：

1. 确定重试范围：
   - 仅物理问题 → 仅重试物理专家 Agent
   - 仅视觉问题 → 仅重试视觉设计 Agent
   - 两者都有 → 同时重试两个 Agent
   - 矛盾检测问题 → 物理专家 Agent 调整规则选择

2. 增量合并：
   - 主 Agent 保留通过方的结果
   - 只替换未通过方的结果
   - 避免"全部推翻重来"导致的震荡

3. 重试终止条件：
   - 3 轮后仍未通过 → 降级
   - 某一方连续 2 轮相同结果 → 标记为"不通过但接受"，降级质量评级
   - 致命矛盾 → 立即终止，拒绝生成

流程图：

验证失败
    │
    ├── 致命矛盾 → 立即拒绝生成
    │
    ├── 物理问题 → 重试物理 Agent（最多3轮）
    │   ├── 通过 → 保留
    │   └── 不通过 → 标记降级
    │
    ├── 视觉问题 → 重试视觉 Agent（最多3轮）
    │   ├── 通过 → 保留
    │   └── 不通过 → 使用通用模板
    │
    └── 两者都有 → 各自重试 + 主 Agent 增量合并
```

```python
# backend/3d/verification/sandbox_simulator.py

class SandboxSimulator:
    """
    沙盒模拟器：在隔离环境中运行物理模拟，验证动画轨迹的物理准确性。
    """
    def simulate(self, physics_spec: PhysicsSpec, duration: float):
        """
        运行数值模拟，返回时间序列数据。
        将 AI 生成的轨迹与数值积分结果对比，差异超过阈值则标记问题。
        """
        trajectory_ai = self.generate_ai_trajectory(physics_spec)
        trajectory_numeric = self.numeric_integration(physics_spec, duration)
        
        errors = []
        for t in range(len(trajectory_ai)):
            diff = abs(trajectory_ai[t] - trajectory_numeric[t])
            if diff > self.threshold:
                errors.append({"time": t, "expected": trajectory_numeric[t], "got": trajectory_ai[t]})
        
        return SimulationResult(
            passed=len(errors) == 0,
            max_error=max(errors, key=lambda e: e["expected"]) if errors else 0,
            error_count=len(errors),
            details=errors[:10]  # 只报告前10个
        )
```

## 6. 物理规则抉择引擎

定义在 `backend/3d/engine/rule_engine_3d.py`，采用 **三层规则抉择机制**，在运行时根据物理状态选择并执行规则。

### 6.1 三层架构

```
物理规则抉择引擎
├── Layer 1: 约束规则（最高优先级）
│   ├── 能量守恒检查（E₁ + W = E₂）
│   ├── 动量守恒检查（m₁v₁ + m₂v₂ = 常矢量）
│   ├── 边界条件强制（s ≥ 0, v ≥ 0, T ≥ 0）
│   └── 单位一致性检查
│
├── Layer 2: 条件规则（状态驱动切换）
│   ├── 摩擦力
│   │   ├── if 相对滑动 → 滑动摩擦力 f = μN
│   │   └── else → 静摩擦力 f ∈ [0, μ₀N]
│   ├── 碰撞
│   │   ├── if 弹性碰撞 → 动量守恒 + 动能守恒
│   │   ├── elif 完全非弹性 → 动量守恒 + 共速
│   │   └── else → 动量守恒 + 恢复系数 e
│   └── 运动状态
│       ├── if 合力为零 → 静止/匀速
│       └── else → 加速运动 F = ma
│
└── Layer 3: 效应规则（由 L2 驱动）
    ├── 合外力 = Σ力 → 加速度
    ├── 加速度积分 → 速度（a = dv/dt）
    ├── 速度积分 → 位移（v = ds/dt）
    └── 功与能 → 能量变化（W = ΔE）
```

### 6.2 执行顺序控制

```python
class RuleExecutionEngine:
    """
    运行时规则抉择引擎。
    执行顺序由 Agent 指定的 execution_order + 规则依赖关系自动推导。
    """
    
    def __init__(self):
        self.layers = {
            1: ConstraintLayer(),    # 约束规则
            2: ConditionalLayer(),   # 条件规则
            3: EffectLayer()         # 效应规则
        }
        self.execution_trace = []    # 记录执行路径（用于可解释性层）
    
    def execute(self, physics_spec: PhysicsSpec, params: dict) -> ExecutionResult:
        """
        按顺序执行三层规则：
        1. 约束规则：先验证物理定律是否满足
        2. 条件规则：根据当前状态选择分支
        3. 效应规则：执行选中的规则，生成运动方程
        
        每步记录执行轨迹，供可解释性层展示。
        """
        # Layer 1: 约束检查
        constraints = self.layers[1].check(physics_spec, params)
        if not constraints.passed:
            return ExecutionResult(failed=True, reason=constraints.violations)
        self.execution_trace.append({"layer": 1, "applied": constraints.applied_rules})
        
        # Layer 2: 条件匹配
        conditions = self.layers[2].evaluate(physics_spec, params)
        self.execution_trace.append({
            "layer": 2, 
            "matched": conditions.matched_rule,
            "condition": conditions.trigger_condition
        })
        
        # Layer 3: 效应执行
        effects = self.layers[3].apply(conditions, params)
        self.execution_trace.append({"layer": 3, "equations": effects.equations})
        
        return ExecutionResult(
            passed=True,
            equations=effects.equations,
            trace=self.execution_trace  # 提供给可解释性层
        )
```

```python
# backend/3d/explainability/causal_labeler.py

class CausalLabeler:
    """
    因果标注生成器。
    根据物理规则执行轨迹，在关键时间点生成"为什么"解释。
    """
    
    def generate_labels(self, execution_trace: list) -> list[dict]:
        """
        从规则执行轨迹生成因果标注。
        
        返回: [
            {"frame": 180, "text": "摩擦力消失，因为相对速度为零", "target": "friction_arrow"},
            {"frame": 360, "text": "动能达到最大值，势能最小", "target": "energy_bar"},
            ...
        ]
        """
        labels = []
        for step in execution_trace:
            if step["layer"] == 2:  # 条件规则切换
                labels.append({
                    "frame": step["frame"],
                    "text": self._explain_condition(step),
                    "target": step.get("affected_object"),
                    "type": "rule_switch",
                    "duration": 120  # 显示 2 秒（60fps）
                })
            elif step["layer"] == 3:  # 效应规则执行
                if "friction" in step.get("applied_rules", []):
                    labels.append({
                        "frame": step["frame"],
                        "text": "摩擦力做负功，机械能减少",
                        "target": "energy_bar",
                        "type": "energy_change",
                        "duration": 90
                    })
        return labels
```

```python
# backend/3d/explainability/comparison_view.py

class ComparisonViewGenerator:
    """
    对比视图生成器。
    生成"实际"vs"假设"两组场景描述，前端分屏渲染。
    """
    
    def generate(self, physics_spec: PhysicsSpec, 
                 visualize_rules: list[str]) -> ComparisonConfig:
        """
        visualize_rules: 需要展示对比效果的规则列表
                        例如 ["friction"] → 有摩擦 vs 无摩擦
        
        返回 ComparisonConfig 包含两组场景参数。
        """
        config = ComparisonConfig()
        
        for rule_id in visualize_rules:
            # 实际场景：正常执行所有规则
            config.add_scene("actual", physics_spec, rules=["all"])
            
            # 假设场景：去掉指定规则
            modified_spec = deepcopy(physics_spec)
            modified_spec.disabled_rules = [rule_id]
            config.add_scene("hypothetical", modified_spec, 
                            rules=[r for r in ALL_RULES if r != rule_id])
            
        return config
```

## 10. 补丁晋升机制

### 10.1 总体流程

```
补丁晋升机制（系统"越用越聪明"的核心）
│
├── 第1层：会话隔离阶段
│   ├── Agent 每次生成的规则/场景组合自动保存为"会话补丁"
│   ├── 补丁只存在于当前会话，不影响其他用户
│   └── 格式：{ "type": "rule"|"scene"|"template",
│               "content": {...},
│               "source_problem": "...",
│               "usage_count": 0 }
│
├── 第2层：临时补丁池
│   ├── 同一补丁被 3+ 个不同题目调用后 → 晋升到临时池
│   ├── 补丁标记为"待观察"，记录使用频率和成功率
│   ├── 定期清理：7 天未使用的补丁自动淘汰
│   └── 格式增加：{ "first_seen": timestamp, "last_used": timestamp, "hit_count": 3 }
│
├── 第3层：晋升投票
│   ├── 触发条件：临时池中补丁被 10+ 次调用且成功率 > 80%
│   ├── 验证官 Agent 自动审查（物理准确性 + 无矛盾）
│   ├── 人类管理员审核（可选，配置化）
│   └── 投票结果：通过 → 固化 | 不通过 → 降级回临时池
│
└── 第4层：固化入库
    ├── 写入规则库（原子规则 / 组合模板 / 场景原语）
    ├── 分配版本号：v{major}.{minor}.{patch}
    ├── 支持回滚：旧版本保留 30 天
    └── 通知各 Agent：规则库已更新
```

### 10.2 补丁数据结构

```python
@dataclass
class Patch:
    """补丁晋升的单位"""
    patch_id: str                        # 补丁 ID
    patch_type: str                      # rule | scene | template | style
    content: dict                        # 补丁内容
    source_problem: str                  # 来源题目
    source_agent: str                    # 来源 Agent
    status: str                          # session | staging | voted | permanent
    hit_count: int = 0                   # 使用次数
    success_count: int = 0               # 成功次数
    fail_count: int = 0                  # 失败次数
    version: str = "0.0.1"              # 版本号
    first_seen: float = None             # 首次出现时间
    last_used: float = None             # 最后使用时间
    created_at: float = None
    approved_by: str = None              # 审批人/Agent
    
    @property
    def success_rate(self) -> float:
        total = self.success_count + self.fail_count
        return self.success_count / total if total > 0 else 0.0
```

### 10.3 晋升条件矩阵

| 晋升阶段 | 触发条件 | 通过条件 | 降级条件 |
|---------|---------|---------|---------|
| session → staging | 同一补丁 3+ 次调用 | 自动 | 7 天未使用 |
| staging → voted | 10+ 次调用且成功率 > 80% | 验证官通过 | 成功率 < 50% |
| voted → permanent | 验证官通过 + 管理员确认（可选） | 写入固化库 | 管理员拒绝 |

### 10.4 补丁管理器

```python
# backend/3d/learning/patch_manager.py

class PatchManager:
    """
    补丁晋升管理器。
    负责补丁的创建、跟踪、晋升、回滚。
    """
    
    def __init__(self):
        self.session_patches: list[Patch] = []   # 会话级
        self.staging_patches: list[Patch] = []   # 临时池
        self.permanent_patches: list[Patch] = [] # 固化库
    
    def record_usage(self, problem_text: str, rules_used: list[str]):
        """记录规则使用情况 → 更新补丁计数"""
        for rule_id in rules_used:
            patch = self._find_patch(rule_id)
            if patch:
                patch.hit_count += 1
                patch.last_used = time.time()
                self._check_promotion(patch)
    
    def _check_promotion(self, patch: Patch):
        """检查补丁是否需要晋升"""
        if patch.status == "session" and patch.hit_count >= 3:
            patch.status = "staging"
        elif patch.status == "staging" and patch.hit_count >= 10 and patch.success_rate > 0.8:
            # 触发晋升投票
            self._request_promotion_vote(patch)
    
    def promote(self, patch_id: str, approved_by: str = "verifier"):
        """执行晋升"""
        patch = self._find_patch(patch_id)
        if not patch:
            return
        
        # 分配版本号
        patch.version = self._next_version(patch)
        patch.status = "permanent"
        patch.approved_by = approved_by
        
        # 写入固化库
        self._write_to_permanent_store(patch)
        
        # 通知各 Agent 规则库已更新
        self._notify_agents(patch)
    
    def rollback(self, patch_id: str):
        """回滚到上一个版本"""
        patch = self._find_permanent(patch_id)
        if patch:
            # 将当前版本标记为废弃
            patch.status = "deprecated"
            # 恢复上一个版本
            prev = self._find_previous_version(patch)
            if prev:
                prev.status = "permanent"
```

```python
        reports = []
        
        # 按题型聚合
        for problem_type in ALL_TYPES:
            type_feedbacks = self._query_feedbacks(problem_type, time_range)
            reports.append({
                "problem_type": problem_type,
                "total": len(type_feedbacks),
                "avg_rating": self._average_rating(type_feedbacks),
                "top_issues": self._top_issues(type_feedbacks, 5),
                "top_praises": self._top_praises(type_feedbacks, 3)
            })
        
        # 生成行动建议
        actions = []
        for report in reports:
            if report["avg_rating"] < 3.0:
                actions.append(f"题型 {report['problem_type']} 评分偏低，建议审查")
            if len(report["top_issues"]) > 0:
                actions.append(f"题型 {report['problem_type']} 高频问题: {report['top_issues'][0]}")
        
        return AggregateReport(reports=reports, suggested_actions=actions)
```

## 12. 异常降级路径

### 12.1 降级路径决策树

```
异常发生
    │
    ├── Agent 超时（>30s）
    │   └── 降级到规则库直出路径（如果有匹配模板）
    │       ├── 有模板 → 直接渲染 3D（降级质量评级）
    │       └── 无模板 → 降级到文字解析
    │
    ├── Agent 置信度 < 40%
    │   └── 降级到文字解析模式
    │       └── 提示"当前无法生成 3D 动画，已为您生成文字解析"
    │
    ├── 验证官连续 3 轮否决
    │   └── 降级到文字解析 + 标记"需要人工审核"
    │       └── 记录失败原因到数据库，供后续分析
    │
    ├── 物理规则库无匹配且 Agent 生成失败
    │   └── 返回"暂不支持该题型" + 建议用户调整描述
    │       └── 记录未支持题型到需求池
    │
    ├── Three.js 渲染失败（浏览器兼容性）
    │   └── 降级到 2D 动画（如果对应 2D 模板存在）
    │       ├── 2D 模板存在 → 渲染 2D 版本
    │       └── 2D 模板不存在 → 文字解析
    │
    ├── LLM 服务不可用
    │   └── 降级到纯规则模式
    │       └── 提示"LLM 服务暂不可用，使用规则模式生成"
    │
    └── 系统内部错误
        └── 返回友好错误页面 + 错误日志
```

## 19. 风险与应对

### 19.1 风险矩阵

| 编号 | 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|------|---------|
| R1 | Three.js 在低端设备性能不足 | 中 | 高 | 分级渲染（LOD + 自适应帧率 + 降级材质） |
| R2 | Agent 调用 LLM 成本高 | 高 | 中 | 80% 规则直出，仅 20% 复杂题启动 Agent |
| R3 | 3D 场景物理不准确 | 中 | 致命 | 验证官 + 矛盾检测库 + 沙盒模拟三重校验 |
| R4 | LLM 幻觉导致错误物理假设 | 中 | 高 | 40% 置信度红线 + 矛盾检测 + 降级策略 |
| R5 | 补丁晋升导致规则库膨胀 | 高 | 中 | 版本管理 + 定期清理 + 回滚机制 |
| R6 | 用户 3D 交互眩晕 | 低 | 中 | 防眩晕模式（平滑过渡 + 速度限制） |
| R7 | 3D 模板维护成本高 | 高 | 中 | 场景原语库抽象 + 声明式组合，非从零编写 |
| R8 | 用户分层导致开发复杂度上升 | 中 | 低 | 分层通过配置驱动，核心逻辑不变 |
| R9 | 可解释性标注影响动画流畅度 | 低 | 低 | 标注渲染独立于主场景，不影响物理动画帧率 |
| R10 | 反馈数据噪声大 | 中 | 低 | 聚合分析 + 高频/高频统计，个别噪声被稀释 |

### 19.2 教学有效性评估方法

```
教学有效性评估
├── 定量评估
│   ├── 前置测试 → 观看动画 → 后置测试（对比得分提升率）
│   ├── 用户停留时间分析（平均观看时长 / 总时长）
│   ├── 交互次数统计（旋转/缩放/点击次数）
│   └── 回看率（拖回进度条重新观看的比例）
│
├── 定性评估
│   ├── 学生主观评分（1-5 分）："这个动画帮助我理解了…"
│   ├── 教师评价："标注清晰度"、"教学逻辑"、"视觉舒适度"
│   └── 自由文本反馈分析
│
└── A/B 测试
    ├── 有可解释性层 vs 无可解释性层
    ├── 不同视觉风格对理解的影响
    └── 新手模式 vs 完整模式的学习效果差异
```

## 附录

### A. 与 2D 模板的差异对比

| 维度 | 2D 模板 | 3D 模板 |
|------|---------|---------|
| 渲染技术 | Canvas 2D | WebGL (Three.js) |
| 场景维度 | 平面（x, y） | 空间（x, y, z） |
| 视角控制 | 无 | OrbitControls |
| 光照系统 | 无 | 环境光 + 方向光 + 点光 |
| 物体模型 | 简单几何 | 3D 几何体 + 材质 + 纹理 |
| 模板构建 | 从零编写 JS | 声明式组合场景原语 |
| 视觉风格 | 固定配色 | 题型自适应风格系统 |
| 用户分层 | 无 | 新手/进阶/竞赛 |
| 可解释性 | 无 | 因果标注 + 对比视图 + 推理路径 |
| 进化能力 | 手动更新 | 补丁晋升机制 |
| 反馈机制 | 无 | 用户反馈闭环 |

### C. 关键术语表

| 术语 | 说明 |
|------|------|
| Agent | 具有特定专业角色的 AI 模块 |
| 场景原语 | 可组合的 Three.js 场景构建块，模板通过原语库声明式组合 |
| 物理规则抉择引擎 | 三层运行时规则执行系统（约束→条件→效应） |
| 矛盾检测库 | 验证官 Agent 加载的规则冲突检测知识库 |
| 补丁晋升 | 临时规则 → 临时池 → 投票 → 固化入库的四层进化机制 |
| 因果标注 | 在关键时间点自动显示"为什么"的教学解释 |
| 对比视图 | 同时显示"实际"和"假设"两组场景的教学工具 |
| OrbitControls | Three.js 轨道控制器 |
| Raycaster | Three.js 射线检测器，用于点击交互 |
| 防眩晕模式 | 相机平滑过渡 + 旋转速度限制的交互保护机制 |
