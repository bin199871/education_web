# 物理教学动画设计理念与实现方案

## 核心理念：让公式长在现象上

### 传统教学 vs 动画教学

| 传统教学 | 理想动画教学 |
|---------|-------------|
| 老师板书：mgh = ½mv² | 学生看到小球从高处滑下，速度越来越快 |
| 老师画受力分析图 | 学生看到力的箭头随运动实时变化 |
| 老师口述：摩擦力做负功 | 学生看到速度数字逐渐归零 |
| 学生背公式 | 学生记住的是 **"现象 + 公式"** |

**最终效果**：学生闭上眼回想时，脑中出现的是小球摆动的画面 + 公式浮现在对应位置，而不是"背了一段文字"。

---

## 现有方案的差距分析

### ✅ 做得好的部分

- 力的箭头随运动实时变化（G、F电、F合）
- 速度箭头 + 数值实时显示
- 能量面板（Ek/Ep_g/Ep_e）动态更新
- 解题步骤逐行淡入展示

### ❌ 三个关键断裂

#### 断裂1：几何量没有在图上标注
摆动到 37° 时，h = 0.2m, x = 0.6m 只出现在解题面板的纯文字里。学生看不到"这个高度差就是h"的视觉对应。

#### 断裂2：能量转化没有视觉因果链
能量面板只显示数字，学生看不到"位置上升 → 势能增加 → 动能减少"的动态连锁反应。

#### 断裂3：解题面板出现时，动画停止了
解题面板出现时画面被半透明遮罩盖住，现象消失了，只剩文字。

## 改进方案

### 1. 几何量可视化标注

当小球摆动到 θ=37° 时，直接在图上画出几何关系：

```
        O
       /|
      / |
     /  |  ← 虚线标注
    /   |  L·sin37° = 0.6m
   /    |
  /θ    |
 ●─────+
   ↑
 L(1-cos37°) = 0.2m
```

用带标签的长度标尺线替代纯数字，公式直接写在标尺旁：
- `h = L(1-cosθ) = 0.2m`
- `x = L·sinθ = 0.6m`

### 2. 能量柱状图代替数字面板

将右下角的数字面板改为动态能量柱：

```
  ┌──────────────┐
  │ 能量柱       │
  │ ▓▓▓▓▓░ Ep_g  │  0.3J
  │ ▓▓▓░░░ Ep_e  │  0.15J
  │ ▓▓████ Ek    │  0.35J
  │ ───────      │
  │ E总 = 0.8J   │  ← 总量不变，柱子变化
  └──────────────┘
```

学生看到柱子在运动，直观理解能量转化，比看三位小数强十倍。

### 3. 解题过程改为"画中画"布局

解题面板不要全屏覆盖，改成从侧边滑入的半透明卡片，动画在小区域继续播放：

```
┌──────────────────────┐
│  • 小球仍在右下角摆动  │
│                    ┌──┤
│                    │  │
│  动画画面继续      │解 │
│                    │题 │
│                    │过 │
│                    │程 │
│                    │  │
└────────────────────┴──┘
```

### 4. 关键瞬态高亮 + 自动慢放

当小球到达 θ=37° 时：
- 画面自动慢放（速度降为 0.3×）
- 标注箭头弹出：h 标注、x 标注、速度箭头放大
- 公式叠加：`W合 = F电·x − G·h = 0.4J` 淡入在对应的几何线上
- 结果闪烁：`v = 2√2 ≈ 2.83m/s` 脉冲放大

7. 将动画数据抽成 JSON 配置

## 设计原则

1. **现象优先**：动画画面永远是主角，文字公式是配角
2. **因果可视化**：每个物理量变化都要有视觉对应
3. **同步呈现**：公式和现象在同一时空出现，不分离
4. **少即是多**：一次只突出一个核心概念，避免信息过载
