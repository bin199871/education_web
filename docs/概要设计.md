## 1. 系统架构

### 1.1 总体架构

采用 **Canvas + HTML DOM 混合架构**：

| 层 | 技术 | 负责内容 |
|----|------|---------|
| 物理动画层 | Canvas 2D | 小球摆动、电场线、受力箭头、角度弧、几何标注线、能量柱状图 |
| 文字覆盖层 | HTML + CSS | 解题面板（画中画）、场景标题、字幕、答案汇总 |
| 控制层 | HTML + CSS + JS | 播放/暂停、进度条、速度切换 |
| 引擎层 | JavaScript | 帧循环、场景调度、物理数据提供 |

**选择混合架构的理由：**
- Canvas 能自由绘制圆弧、箭头、粒子效果，适合物理可视化
- HTML/DOM 天然支持复杂文字排版、数学公式（可通过 Unicode/HTML 实体实现）、CSS 过渡动画
- 解题面板的文字内容独立于 Canvas，便于维护和修改

## 2. 模块设计

### 2.1 模块职责与接口

| 模块 | 职责 | 对外接口 |
|------|------|---------|
| **FrameController** | 驱动 requestAnimationFrame 循环，维护帧计数器，处理播放/暂停/调速/跳转 | `play()`, `pause()`, `seek(frame)`, `setSpeed(s)`, `getCurrentFrame()` |
| **SceneDispatcher** | 根据当前帧号确定场景，触发场景进入/退出事件 | `getScene(frame)` → sceneId, `onEnter(sceneId)`, `onExit(sceneId)` |
| **PhysicsData** | 预计算两段释放过程的物理数据，提供帧→物理量的查询 | `getPhysics(sceneId, progress)` → `{theta, v, Ek, Ep_g, Ep_e, Etot, x, h}` |
| **CanvasRenderer** | 管理 Canvas 上所有图形元素的绘制 | `render(ctx, frame, scene, physics)` |
| **SolutionPanel** | 管理解题面板的 DOM 元素、文本内容和动画 | `show(sceneId)`, `hide()`, `advanceStep(stepIndex)` |
| **UIController** | 绑定控制栏事件、键盘事件、更新进度和时间显示 | 事件监听 |

### 2.2 FrameController 设计

```
FrameController
├── 状态: { frame, totalFrames, isPlaying, speed, fps }
├── 方法:
│   ├── start()           → 启动动画循环
│   ├── pause()           → 暂停
│   ├── resume()          → 继续
│   ├── seek(targetFrame) → 跳转到指定帧
│   ├── setSpeed(s)       → 设置播放速度 (0.5/1/1.5/2)
│   └── reset()           → 回到开头
├── 事件回调:
│   ├── onFrame(frame)    → 每帧回调，驱动渲染
│   └── onComplete()      → 播放完成回调
└── 内部循环:
    requestAnimationFrame → 计算deltaTime → 累加frame → 调用onFrame
```

核心逻辑伪代码：

```javascript
function animationLoop(timestamp) {
  if (!lastTimestamp) lastTimestamp = timestamp;
  const delta = (timestamp - lastTimestamp) / 1000;
  lastTimestamp = timestamp;

  if (isPlaying) {
    const frameDelta = delta * fps * speed;
    currentFrame = Math.min(currentFrame + frameDelta, totalFrames - 1);
    onFrame(Math.floor(currentFrame));
    
    if (currentFrame >= totalFrames - 1) {
      isPlaying = false;
      onComplete();
    }
  }
  requestAnimationFrame(animationLoop);
}
```

核心方法：

```javascript
function getScene(frame, totalFrames) {
  const pct = frame / totalFrames;
  for (const s of SCENES) {
    if (pct >= s.start && pct < s.end) return s.id;
  }
  return SCENES[SCENES.length - 1].id;
}

function getSceneProgress(frame, sceneId, totalFrames) {
  const s = SCENES.find(x => x.id === sceneId);
  const pct = frame / totalFrames;
  return (pct - s.start) / (s.end - s.start);  // 0~1
}
```

### 2.4 PhysicsData 设计

**物理模型：**

复合场中单摆的势能（以 A 点为零势能点）：
```
U(θ) = mg·L(1-cosθ) - qE·L·sinθ
```

速度由能量守恒求得：
```
v(θ) = √(2·(qE·L·sinθ - mg·L·(1-cosθ)) / m)
```

**两段运动数据预计算：**

| 数据集 | 释放位置 | θ范围 | 帧数 | 用途 |
|--------|---------|-------|------|------|
| `data_swing1` | A点(θ=0°) | 0° → 90°（最大值） | 120帧 | Q2摆动过程 |
| `data_swing2` | 水平(θ=90°) | 90° → 0° | 120帧 | Q3摆动过程 |

预计算数据结构：

```javascript
function computeSwingData(thetaStart, thetaEnd, numSteps) {
  const data = [];
  for (let i = 0; i <= numSteps; i++) {
    const theta = thetaStart + (thetaEnd - thetaStart) * (i / numSteps);
    const sinT = Math.sin(theta);
    const cosT = Math.cos(theta);
    const h = L * (1 - cosT);          // 高度差
    const x = L * sinT;                // 水平位移
    const Ek = qE * L * sinT - m*g * L * (1 - cosT);
    const v = Ek > 0 ? Math.sqrt(2 * Ek / m) : 0;
    const Ep_g = m*g * h;
    const Ep_e = -qE * x;  // 电场力做正功，电势能减少
    data.push({
      theta, h, x, v,
      Ek: 0.5*m*v*v,
      Ep_g, Ep_e,
      Etot: 0.5*m*v*v + Ep_g + Ep_e
    });
  }
  return data;
}
```

**实时查询：**

```javascript
function getPhysics(sceneId, progress) {
  // progress: 0~1 表示在当前场景的进度
  const dataset = (sceneId === 'swing_q2' || sceneId === 'highlight') 
    ? data_swing1 
    : data_swing2;
  const idx = Math.min(Math.floor(progress * dataset.length), dataset.length - 1);
  return dataset[idx];
}
```

```

Layer 2: 电场线 (虚线+箭头动画)
Layer 3: 天花板+悬点+圆弧轨迹
Layer 4: 细线+小球
Layer 5: 角度弧+标注

Layer 7: 受力箭头 (G, F电, F合, T)

Layer 10: 场景标题/字幕

    this.drawEnergyBars(physics);      // Layer 9
    this.drawSceneTitle(scene);        // Layer 10

  // 各绘制方法详细设计...
}
```

## 7. 两段释放的动画实现细节

这是本设计区别于传统实现的关键创新点。

### 7.1 第一次释放（Q1/Q2）

**视觉流程：**
1. 小球静止在 A 点（θ=0°）
2. Q1 受力分析：G + F电 + F合 箭头依次弹出
3. 过渡后，小球从 A 点向右摆动
4. θ 实时显示，到达 37° 时慢放 + 标注

**物理数据源：** `data_swing1`（θ: 0° → 最大角）

**场景映射到摆动进度：**

```javascript
// swing_q2 场景: 0→1 映射到摆动全程
// highlight 场景: 0→1 映射到 θ=37° 附近的慢放区间
function getSwingProgress(sceneId, sceneProgress) {
  if (sceneId === 'swing_q2') return sceneProgress;
  if (sceneId === 'highlight') return 0.45 + sceneProgress * 0.1;  // 37°附近
  return 0;
}
```

### 7.2 第二次释放（Q3）

**视觉流程：**
1. 小球被"拉"到水平位置（θ=90°）
2. 标注："将小球拉到水平位置由静止释放"
3. 释放，小球摆向 A 点
4. 到达 A 点时速度最大，显示 T、G、F电
5. Tmax = 5.0 N 高亮

**物理数据源：** `data_swing2`（θ: 90° → 0°）

**拉球动画：** 在 `pull_90` 场景中，θ 从 0° 线性增加到 90°，模拟拉球过程。

### 7.3 关键帧慢放实现

到达 θ=37° 时自动减速：

```javascript
// 在 FrameController 中
const SLOW_MOTION_EVENTS = [
  { start: highlightStart, end: highlightEnd, speed: 0.3 }
];

function getEffectiveSpeed(frame) {
  for (const sm of SLOW_MOTION_EVENTS) {
    if (frame >= sm.start && frame < sm.end) {
      return this.speed * sm.speed;  // 在用户设定速度基础上再降速
    }
  }
  return this.speed;
}
```

## 10. 技术决策记录

| 决策 | 选项 | 选择 | 理由 |
|------|------|------|------|
| 渲染方式 | 纯Canvas / 纯DOM / 混合 | **混合** | 物理绘制用Canvas，文字排版用DOM |
| 帧率 | 30fps / 60fps | **60fps** | 物理动画需要流畅 |
| 总时长 | 2:00 / 2:30 / 3:00 | **3:00** | 12个场景需要足够时间 |
| 总帧数 | — | **10800** | 3min × 60fps |
| 物理计算 | 实时计算 / 预计算 | **预计算** | 保证每帧性能稳定，方便调试 |
| 解题面板 | Canvas绘制 / DOM | **DOM** | 文字排版、CSS过渡更灵活 |
| 响应式 | 固定 / 自适应 | **固定960×640** | 物理教学场景有固定尺寸，保证视觉效果 |

## 12. 开发建议

### 12.1 实现顺序建议

| 优先级 | 模块 | 预估工作量 |
|--------|------|-----------|
| P0 | 物理数据预计算 + 基础 Canvas 渲染（背景/小球/细线） | 2h |
| P0 | 帧循环 + 播放/暂停控制 | 1h |
| P1 | 受力箭头绘制 | 1h |
| P1 | 角度弧 + 几何标注线 | 1.5h |
| P1 | 解题面板 DOM + 步骤展示 | 2h |
| P2 | 能量柱状图 | 1h |
| P2 | 速度箭头 | 0.5h |
| P2 | 场景调度 + 过渡特效 | 1h |
| P3 | 关键帧慢放 | 0.5h |
| P3 | 进度条/速度切换/快捷键 | 1h |
| P3 | 答案汇总页 | 0.5h |

### 12.2 增量迭代策略

每次迭代后播放一次完整动画，验证效果：

1. **Iter 1**：物理数据 + 小球摆动（裸场景）
2. **Iter 2**：加入场景调度 + 两段释放切换
3. **Iter 3**：加入受力箭头 + 角度弧 + 标注
4. **Iter 4**：加入解题面板 + 能量柱状图
5. **Iter 5**：加入特效（慢放、过渡动画）+ 控制栏
