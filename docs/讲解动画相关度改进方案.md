# 讲解动画相关度改进方案

## 当前问题

讲解动画的架构是 **1 模板 = 1 固定题型 + 固定场景序列**。模板手写，场景和时间分配不可变。

```
题目: "光滑斜面 m=2kg, θ=37°, L=3m, 求底端速度"
       ↓ 匹配 inclined_plane 模板
       ↓ 播 10 个场景（含粗糙面相关 x4）
       ↓ 相关度 ≈ 50%
```

```
题目: "电场单摆 m=0.1, q=5e-4, E=2000, L=1, 求 F电/F合/v@37°"
       ↓ 匹配 electric_pendulum 模板
       ↓ 播 12 个场景（全对齐）
       ↓ 相关度 ≈ 95%
```

根源：**场景结构硬编码**，不对题目的「求什么」做裁剪。

---

## 改进方案：三阶段

### 阶段一：场景裁剪（基于 `phases` 检测）

#### 思路

`extract_physics_params()` 的 `phases` 数组已经给出了题目包含哪些物理过程：

```python
# "光滑斜面求速度"
phases = [{"type": "slope", "params": {...}}]  # 无粗糙面

# "斜面+粗糙面+拉力"
phases = [
    {"type": "slope", "params": {...}},
    {"type": "rough_surface", "params": {...}},
    {"type": "horizontal_pull", "params": {...}},
]
```

模板的 `SCENES` 数组可以在运行时根据 `phases` 做选择性的启用/跳过。

#### 具体做法

给每个场景加一个 `requires` 条件，运行时判断是否启用：

```js
// inclined_plane 模板
const SCENES = [
  {id: 'intro',     s:0.00, e:.08, requires: '*'},              // 总是显示
  {id: 'forces',    s:.08,  e:.20, requires: '*'},              // 总是显示
  {id: 'slide_sm',  s:.20,  e:.35, requires: 'has_slope'},      // 有斜面时
  {id: 'hilight',   s:.35,  e:.42, requires: 'has_slope'},      // 有斜面时
  {id: 'sol_q1',    s:.42,  e:.52, requires: 'has_slope'},      // 有斜面时
  {id: 'trans',     s:.52,  e:.58, requires: 'has_rough'},      // 有粗糙面时
  {id: 'slide_rh',  s:.58,  e:.72, requires: 'has_rough'},      // 有粗糙面时
  {id: 'sol_q2',    s:.72,  e:.82, requires: 'has_rough'},      // 有粗糙面时
  {id: 'sol_q3',    s:.82,  e:.90, requires: 'has_rough'},      // 有粗糙面时
  {id: 'ending',    s:.90,  e:1.0, requires: '*'},              // 总是显示
];
```

运行时根据 `__INJECTED_PARAMS__` 中的 `_phases` 判断哪些场景激活，然后重新映射时间分配：

```js
// 初始化时：计算激活场景列表 + 重新分配时间
const _phases = _IN._phases || ['slope', 'rough'];
const hasSlope = _phases.includes('slope');
const hasRough = _phases.includes('rough_surface');

// 过滤出激活的场景
const ACTIVE_SCENES = SCENES.filter(s =>
  s.requires === '*' ||
  (s.requires === 'has_slope' && hasSlope) ||
  (s.requires === 'has_rough' && hasRough)
);

// 重新分配时间（等比例拉伸到 0→1）
const totalWeight = ACTIVE_SCENES.reduce((sum, s) => sum + (s.e - s.s), 0);
let cursor = 0;
ACTIVE_SCENES.forEach(s => {
  const weight = s.e - s.s;
  s.s = cursor;
  s.e = cursor + weight / totalWeight;
  cursor = s.e;
});
```

#### 效果对比

改造前（无论什么题都播 10 个场景）：

```
光滑面题:  intro → forces → slide_sm → hilight → sol_q1
          → trans → slide_rh → sol_q2 → sol_q3 → ending
            ↑ 粗糙面相关场景 4 个，与题目无关 ↑
```

改造后（根据 phases 自动裁剪）：

```
光滑面题:  intro → forces → slide_sm → hilight → sol_q1 → ending   ✅
斜面+粗糙: intro → forces → slide_sm → hilight → sol_q1
          → trans → slide_rh → sol_q2 → sol_q3 → ending            ✅
全部:     intro → forces → slide_sm → hilight → sol_q1
          → trans → slide_rh → sol_q2 → sol_q3
          → pull → sol_q4 → ending                                  ✅
```

#### 注入参数新增

`template_generate()` 将 `phases` 也传入模板：

```json
__INJECTED_PARAMS__ = {
  "m": 2.0, "theta": 37, "L": 3.0, "mu": 0.4, "g": 10,
  "_phases": ["slope"]            // 只有斜面，没有粗糙面
}
```

### 阶段二：解题步骤数值动态化

#### 思路

当前步骤文本是静态 HTML：

```html
<div class="step-row" id="s1s1">
  <span>= √(2×6×3) = √36</span>         ← 6 和 3 来自默认参数
</div>
```

改为 JS 动态计算后填充：

```js
const _STEPS = {
  sol_q1: [
    `a = g·sinθ = ${P.g}×${P.sin.toFixed(1)} = ${P.a_smooth.toFixed(1)} m/s²`,
    `v² = 2aL → v = √(2×${P.a_smooth.toFixed(1)}×${P.L}) = √${(2*P.a_smooth*P.L).toFixed(0)}`,
    `v = ${P.v_smooth.toFixed(2)} m/s`,
  ],
};

function renderSteps(solId, steps) {
  steps.forEach((text, i) => {
    const el = document.getElementById(solId + 's' + i);
    if (el) el.innerHTML = '<span>' + text + '</span>';
  });
}
```

#### 影响范围

| 模板 | 步骤数 | 需要改写的 HTML 行数 |
|---|---|---|
| `electric_pendulum.html` | 3 组 × 8 步 = 24 步 | ~70 行（步骤 HTML + JS 模板） |
| `inclined_plane.html` | 3 组 × 2-3 步 = 8 步 | ~40 行 |
| `projectile.html` | 3 组 × 2-3 步 = 8 步 | ~35 行 |
| `vertical_circular.html` | 3 组 × 3 步 = 9 步 | ~35 行 |
| `conveyor_belt.html` | 无解题面板 | 0 |
| `board_block.html` | 无解题面板 | 0 |

#### 难度评估

**中等。** 每步的 HTML 需要从 `<span>硬编码</span>` 改为 `<span id="dynamic-s1">占位</span>` + JS 模板填充。步骤中的公式需要保持可读性（保留 `√`, `²` 等字符）。

### 阶段三：场景模板 + 目标驱动的动态编排

#### 思路

将场景视为**可组合的积木**。不是整个模板一起渲染，而是：

1. 检测题目问的是什么（从文本中提取问题目标）
2. 从场景库中选择相关的场景片段
3. 拼接成完整的播放序列

#### 问题目标提取

```python
# 从题目文本中提取"求什么"
def extract_goals(text):
    """返回目标列表，如 ['velocity', 'force', 'tension']"""
    goals = []
    # "求物块在水平面上滑行的最大距离" → 'displacement'
    # "求小球落地时的速度" → 'velocity'
    # "求电场力的大小和合力" → 'force'
    # 正则匹配 "求.*?(速度|力|加速度|时间|距离|位移|功|能|功率)"
    return goals
```

#### 场景积木库

每个模板按场景拆分为独立片段，每个片段标注解决的问题目标和依赖的物理过程：

```python
SCENE_BRICKS = {
    'inclined_plane': [
        {
            'id': 'intro',
            'label': '题目介绍',
            'duration': 0.08,
            'goals': [],               # 通用，无特定目标
            'requires_phases': [],     # 总是显示
        },
        {
            'id': 'forces',
            'label': '受力分析',
            'duration': 0.12,
            'goals': ['force'],
            'requires_phases': ['slope'],
        },
        {
            'id': 'slide_smooth',
            'label': '光滑面加速下滑',
            'duration': 0.15,
            'goals': ['velocity', 'acceleration', 'time'],
            'requires_phases': ['slope'],
            'requires_rough': False,   # 光滑面
        },
        {
            'id': 'slide_rough',
            'label': '粗糙面减速',
            'duration': 0.14,
            'goals': ['velocity', 'displacement', 'force'],
            'requires_phases': ['rough_surface'],
        },
        {
            'id': 'sol_velocity',
            'label': '求速度',
            'duration': 0.10,
            'goals': ['velocity'],
            'requires_phases': ['slope'],
        },
        {
            'id': 'sol_force',
            'label': '求摩擦力',
            'duration': 0.10,
            'goals': ['force'],
            'requires_phases': ['rough_surface'],
        },
        {
            'id': 'sol_displacement',
            'label': '求位移',
            'duration': 0.08,
            'goals': ['displacement'],
            'requires_phases': ['rough_surface'],
        },
    ],
}
```

#### 编排算法

```python
def compose_scenes(template_id, phases, goals):
    """根据物理过程和问题目标，从积木库中选择合适的场景并编排。"""
    bricks = SCENE_BRICKS[template_id]

    selected = []

    # 1. 强制添加通用场景（intro, ending）
    selected.append(next(b for b in bricks if b['id'] == 'intro'))
    selected.append(next(b for b in bricks if b['id'] == 'forces'))

    # 2. 根据 phases 添加过程展示场景
    has_slope = 'slope' in [p['type'] for p in phases]
    has_rough = 'rough_surface' in [p['type'] for p in phases]

    if has_slope and not has_rough:
        selected.append(next(b for b in bricks if b['id'] == 'slide_smooth'))
    elif has_slope and has_rough:
        selected.append(next(b for b in bricks if b['id'] == 'slide_smooth'))
        selected.append(next(b for b in bricks if b['id'] == 'slide_rough'))

    # 3. 根据问题目标添加解答场景
    for goal in goals:
        solution_bricks = [b for b in bricks
                          if goal in b['goals']
                          and b['id'].startswith('sol_')]
        selected.extend(solution_bricks)

    # 4. 总是以 ending 收尾
    selected.append(next(b for b in bricks if b['id'] == 'ending'))

    # 5. 去重（保留顺序）
    seen = set()
    unique = []
    for b in selected:
        if b['id'] not in seen:
            seen.add(b['id'])
            unique.append(b)

    # 6. 重新分配时间
    total_dur = sum(b['duration'] for b in unique)
    cursor = 0.0
    for b in unique:
        b['s'] = cursor
        cursor += b['duration'] / total_dur
        b['e'] = cursor

    return unique
```

编排结果（6场景）:
[0-14%]  intro
[14-35%] forces

[58-76%] sol_velocity  (求速度解答)

## 实现路线图

| 阶段 | 内容 | 前置依赖 | 预估工期 |
|---|---|---|---|
| **1.1** | 给 `SCENES` 加 `requires` 条件 + 运行时裁剪逻辑 | 已完成参数注入 | 1-2天 |
| **1.2** | 后端 `template_generate()` 传入 `_phases` | 已完成 `extract_physics_params` | 0.5天 |
| **2.1** | 每个模板的步骤文本改为 JS 动态填充 | 阶段 1 | 2-3天/模板 |
| **2.2** | 答案汇总面板 `PARAMS.questions.answer` 动态化 | 阶段 2.1 | 0.5天/模板 |
| **3.1** | `extract_goals()` 问题目标提取 | 无 | 1天 |
| **3.2** | `SCENE_BRICKS` 场景积木库定义 | 阶段 1 | 1天 |
| **3.3** | `compose_scenes()` 编排引擎 | 阶段 3.1+3.2 | 2天 |
| **3.4** | 模板渲染器改为接收编排后的场景列表 | 阶段 3.3 | 1天 |

### 优先级建议

**先做阶段一**（场景裁剪），收益最大、改动最小。核心逻辑在 JS 层面，只改 `SCENES` 过滤和重分配时间，不涉及渲染逻辑。

**阶段二**（步骤数值动态化）适合对部分高使用率模板做（如 `electric_pendulum`、`inclined_plane`），其余模板可延后。

**阶段三**（积木编排）是架构级改造，需要定义 `SCENE_BRICKS` 库和编排器，但长期来看是让模板真正适配题目的关键。

## 不处理的（本次范围外）

| 事项 | 原因 |
|---|---|
| 自动生成全新场景（非积木拼接） | 需要 LLM + Canvas 渲染器生成，属独立课题 |
| 步骤文本中的公式排版优化 | LaTeX 渲染引擎，与相关度无关 |
| 场景间平滑过渡的自动生成 | 视觉品质问题，非相关度 |
