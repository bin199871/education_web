# 物理动画模板系统 · 需求文档

> 基于 physics-pendulum.html 的设计质量，将"现象+公式"的教学理念推广到更多题型

---

## 1. 项目愿景

**一句话目标**：输入任意一道高中物理大题 → 自动生成一个 physics-pendulum.html 品质的讲解动画 HTML 文件。

### 1.1 核心价值

| 当前状态 | 目标状态 |
|---------|---------|
| 每个动画手工编码，耗时2-3天 | 输入题目，自动生成 |
| 仅支持电场单摆一种题型 | 支持15-20种高频题型 |
| 动画代码与题目数据耦合 | 模板与数据分离，参数注入 |
| 只有开发者能创建新动画 | 定义好模板就能扩展题型 |

### 1.2 设计理念继承

所有模板必须遵循 physics-animation-design-philosophy.md 中的四条原则：

1. **现象优先** — 动画画面永远是主角，文字公式是配角
2. **因果可视化** — 每个物理量变化都要有视觉对应
3. **同步呈现** — 公式和现象在同一时空出现，不分离
4. **少即是多** — 一次只突出一个核心概念

### 2.2 斜面 + 摩擦力（新做）

**典型题目：**
> 一个质量 m = 2 kg 的物体从倾角 θ = 37° 的光滑斜面顶端由静止滑下，斜面长度 L = 3 m，求物体滑到底端时的速度。若斜面粗糙，动摩擦因数 μ = 0.2，求摩擦力做功和滑到底端的速度。

**场景分镜（12场景）：**

| 场景 | 内容 |
|------|------|
| 开场 | 斜面 + 物体 + 已知量标注 |
| 受力分析 | G分解为G∥和G⊥，支持力N |
| 情况1：光滑面 | 物体下滑，速度越来越快 |
| θ高亮 | 角度标注，L标注，h = L·sinθ |
| 能量分析 | 重力势能→动能，柱状图 |
| 解题1 | 动能定理求v |
| 过渡 | "下面考虑粗糙情况" |
| 情况2：粗糙面 | 增加摩擦力f = μN |
| 受力对比 | 光滑 vs 粗糙 受力对比 |
| 能量对比 | 摩擦力做功 → 热能 |
| 解题2 | 考虑摩擦的动能定理 |
| 答案汇总 | 两种情况对比 |

**参数：**
```json
{
  "m": 2.0,
  "theta": 37,
  "L": 3.0,
  "mu": 0.2,
  "g": 10,
  "hasFriction": true,
  "question1": "求光滑面滑到底端的速度",
  "question2": "求粗糙面摩擦力做功和速度",
  "answer1": "v = 6 m/s",
  "answer2": "v ≈ 4.9 m/s, Wf = -6 J"
}
```

| 位移标注 | 水平射程 x、竖直位移 y |
| 解题1 | 求落地时间 |
| 解题2 | 求水平射程 |
| 解题3 | 求落地速度 |

## 3. 模板格式规范

### 3.1 模板文件结构

每个模板是一个**自包含的独立 HTML 文件**：

```
templates/
  electric_pendulum.html    ← 电场单摆
  inclined_plane.html       ← 斜面+摩擦
  projectile.html           ← 平抛
  circular_motion.html      ← 圆周运动（后续）
  spring.html               ← 弹簧振子（后续）
  collision.html            ← 碰撞（后续）
  ...
```

### 3.2 模板内部规范

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>[[TITLE]] - 物理讲解动画</title>
  <style>
    /* 所有样式内联 */
  </style>
</head>
<body>
  <!-- DOM 结构 -->
  <script>
    // ============================================================
    //  【1】参数注入区（由后端替换）
    // ============================================================
    var TEMPLATE_PARAMS = null;  // ← 后端注入此变量
    // ============================================================

    // 合并默认参数与注入参数
    var PHYS = Object.assign({
      // 默认值（用于独立预览）
      m: 1.0,
      g: 10,
      // ... 题型相关默认值
      problem: '示例题目',
      questions: [
        { title: '第(1)问', answer: '答案1' },
        { title: '第(2)问', answer: '答案2' },
        { title: '第(3)问', answer: '答案3' }
      ]
    }, TEMPLATE_PARAMS || {});

    // ============================================================
    //  【2】动画引擎（模板主体代码）
    // ============================================================
    // ... 帧控制器、场景调度、Canvas渲染、解题面板等

    // ============================================================
    //  【3】启动
    // ============================================================
  </script>
</body>
</html>
```

### 3.3 参数注入机制

后端读取模板文件，将 `TEMPLATE_PARAMS` 占位替换为实际 JSON：

```python
import json, re

def render_template(template_path, params):
    with open(template_path, 'r', encoding='utf-8') as f:
        html = f.read()

    # 注入参数
    param_json = json.dumps(params, ensure_ascii=False)
    html = html.replace(
        'var TEMPLATE_PARAMS = null;',
        f'var TEMPLATE_PARAMS = {param_json};'
    )

    # 注入标题
    html = html.replace('[[TITLE]]', params.get('title', '物理讲解动画'))

    return html
```

### 3.4 模板自洽性要求

每个模板必须满足：

| 要求 | 说明 |
|------|------|
| **独立运行** | 不注入参数时，使用默认值也能播放演示动画 |
| **12场景结构** | 不低于10个场景的教育节奏（开场→受力→解题→过渡→过程→高亮→解题→过渡→过程→解题→汇总） |
| **参数驱动** | 所有物理量、题目文本、答案文本均来自参数 |
| **颜色编码** | 受力箭头严格遵循颜色规范（G蓝/F电红/合橙/T紫/v青） |
| **画中画面板** | 解题面板从右侧滑入，不遮挡主画面 |
| **能量可视化** | 有能量转化过程的题型必须包含柱状图 |
| **几何标注** | 关键位置必须有几何量标注线 |

### 4.2 template_engine.py 设计

```python
import json
import re
from pathlib import Path
from typing import Optional

class TemplateEngine:
    """
    物理动画模板引擎。
    负责：模板匹配、参数注入、HTML生成
    """

    # 模板注册表：题型标识 → 模板文件 + 所需参数
    REGISTRY = {
        'electric_pendulum': {
            'file': 'electric_pendulum.html',
            'label': '电场中带电单摆',
            'params': ['m', 'q', 'E', 'L', 'g', 'theta_target'],
            'tags': ['电场', '单摆', '复合场', '圆周运动']
        },
        'inclined_plane': {
            'file': 'inclined_plane.html',
            'label': '斜面 + 摩擦力',
            'params': ['m', 'theta', 'L', 'mu', 'g', 'v0'],
            'tags': ['斜面', '摩擦力', '牛顿定律', '能量']
        },
        'projectile': {
            'file': 'projectile.html',
            'label': '平抛运动',
            'params': ['v0', 'h', 'g'],
            'tags': ['平抛', '曲线运动', '运动的合成与分解']
        }
    }

    def __init__(self, templates_dir: str = None):
        self.templates_dir = templates_dir or (
            Path(__file__).parent / 'templates'
        )

    def list_templates(self) -> list:
        """返回所有可用模板列表"""
        return [
            {'id': k, **v}
            for k, v in self.REGISTRY.items()
        ]

    def match_template(self, analysis: dict) -> Optional[str]:
        """
        根据 Layer1 分析结果匹配最佳模板。
        返回模板ID，匹配不到返回 None。
        """
        topic = analysis.get('topic', '')
        tags = analysis.get('tags', [])

        # 评分制匹配
        scores = []
        for tid, tpl in self.REGISTRY.items():
            score = 0
            # 标签匹配
            for tag in tpl['tags']:
                if tag in tags or tag in topic:
                    score += 10
            # 关键词匹配
            text = (analysis.get('text', '') + topic).lower()
            for tag in tpl['tags']:
                if tag in text:
                    score += 5
            scores.append((tid, score))

        if not scores:
            return None

        # 取最高分
        best = max(scores, key=lambda x: x[1])
        return best[0] if best[1] > 0 else None

    def extract_params_via_llm(self, problem_text: str, template_id: str) -> dict:
        """
        调用 LLM 从题目文本中提取模板所需的参数。
        返回参数dict。
        """
        tpl = self.REGISTRY.get(template_id)
        if not tpl:
            raise ValueError(f'未知模板: {template_id}')

        # 构建 prompt
        prompt = self._build_extraction_prompt(problem_text, tpl)

        # 调用 LLM（通过现有的 layer1_agent 的 LLM 接口）
        # ...

    def _build_extraction_prompt(self, text: str, tpl: dict) -> str:
        return f"""你是一个物理题目参数提取器。请从以下题目中提取 {tpl['label']} 相关的物理参数。

题目文本：
{text}

需要提取的参数（JSON格式）：
{json.dumps({p: '' for p in tpl['params']}, ensure_ascii=False, indent=2)}

注意：
- 数值使用 float 类型
- 如果题目中未明确给出，使用常识默认值（如 g=10）
- 输出格式：纯 JSON，不要额外说明
- 角度用度（°）为单位

示例输出：{{"m": 0.1, "q": 5e-4, "E": 2000, "L": 1.0, "g": 10, "theta_target": 37}}"""

    def render(self, template_id: str, params: dict) -> str:
        """渲染模板，返回完整 HTML"""
        tpl = self.REGISTRY.get(template_id)
        if not tpl:
            raise ValueError(f'未知模板: {template_id}')

        template_path = self.templates_dir / tpl['file']
        return render_template(str(template_path), params)

### 4.4 LLM 参数提取 Prompt 详细设计

LLM 需要从原始题目文本中提取结构化参数。这是决定生成质量的关键环节。

**Prompt 结构：**

```
你是一个高中物理题参数提取器。

题目文本：
{problem}

题型：{template_label}

请从题目中提取以下参数，输出为 JSON：
{required_params_schema}

参数说明：
{param_descriptions}

输出规则：
- 数值用 float，角度用度（°）
- 有量纲的数值不要带单位
- 缺失参数：如果题目明确给出则用题目值，否则用标准默认值
- 科学计数法：用 e 格式（如 5e-4）
- 只输出 JSON，不要多余文字
```

**Param descriptions example for electric_pendulum:**
```
- m: 小球质量 (kg)
- q: 小球电荷量 (C)
- E: 电场强度 (N/C)
- L: 细线长度 (m)
- g: 重力加速度 (m/s²)，默认 10
- theta_target: 目标角度 (°)，默认 37
- question1: 第(1)问的题目标题
- question2: 第(2)问的题目标题
- question3: 第(3)问的题目标题
- answer1: 第(1)问的答案
- answer2: 第(2)问的答案
- answer3: 第(3)问的答案
```

### 5.1 完整流程

```
用户输入题目文本
       │
       ▼
Layer1: 题目分析 ───→ 提取 topic、tags、key concepts
       │
       ▼
Template Engine: 匹配模板
       │

## 6. 模板创建指南（供后续扩展用）

### 6.1 创建新模板的步骤

1. **分析题目类型** — 确定物理过程、场景分镜、需要标注的几何量
2. **定义参数 schema** — 确定所有可配置的物理量和题目文本
3. **复制参考模板** — 基于 electric_pendulum.html 改造
4. **调整场景分镜** — 修改 SCENE_DEFS 适应新题型
5. **替换物理引擎** — 实现新题型的物理数据预计算
6. **替换 Canvas 绘制** — 绘制新题型的视觉元素
7. **替换解题面板内容** — 更新 PANEL_CONTENTS
8. **注册到 REGISTRY** — 在 template_engine.py 中注册
9. **测试** — 用默认参数独立运行，用真实题目参数注入测试

## 9. 技术决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 模板格式 | 独立 HTML 单文件 | 可独立运行、可下载、可分享 |
| 参数注入 | 后端字符串替换 | 简单可靠，无运行时依赖 |
| 参数提取 | LLM 全权负责 | 正则方案维护成本高，LLM 准确度够用 |
| 模板匹配 | 标签评分制 | 简单可预测，方便调试 |
| 默认值 | 每个模板自带 | 独立预览时不依赖后端 |
| 开发顺序 | 先框架→后内容 | 框架稳定后再批量生产模板 |
