# 剩余 14 种题型积木化改动规划

## 总体策略

分三级难度推进：

| 级别 | 含义 | 数量 | 预估工期 |
|---|---|---|---|
| **A-改造** | 已有模板文件，只需套 brick 架构 | 3 | 1天/个 |
| **B-修复** | 模板文件存在但损坏，需修复+改造 | 2 | 1.5天/个 |
| **C-新建** | 无模板文件，需从零创建 | 9 | 1-3天/个 |

### 优先级

```
P0:  projectile, vertical_circular    (已有高质量模板，快速收割)
P1:  conveyor_belt, board_block       (修复现有资产)
P2:  spring_oscillator, collision     (高频考点，物理简单)
P3:  connected_bodies, locomotive     (中频考点)
P4:  magnetic_deflection, conductor_cutting, circuit_analysis, astronomy, mechanical_wave, gas_law
```

### A1. `projectile.html` — 平抛运动

**现状**：有完整模板文件（18KB），参数注入已做，已知面板动态。但渲染用旧 sceneId switch。

**物理模型**：

**Goal 映射**：

```python
# 已在 server.py GOAL_PATTERNS 中
'velocity':       [r'(?<!最低点)(?<!底端)(?<!末端)(?<!A点)速度(?!.*最低点)(?!.*底端)']

'landing_velocity':[r'落地速度', r'末速度', r'着地速度']
```

**Bricks（5块）**：
```
intro       — 题目介绍（已知量）
launch      — 水平抛出动画（球从平台水平飞出）
flight      — 飞行过程（parabola轨迹+速度分解vx/vy）
sol_time    — 求落地时间 [需要 physics + steps]
```

```js
// 根据进度返回物理状态
function flightData(prog) {
  var t = prog * P.tFall * 1.2;  // 略微延长展示时间
  var x = P.v0 * t;
  var y = 0.5 * P.g * t * t;

```

**步骤模板**：

### A2. `vertical_circular.html` — 竖直圆周运动

**现状**：完整模板文件（16KB），参数注入已做，已知面板动态。

**物理模型**：
```
最高点：N₁ + G = mv₁²/R → N₁ = mv₁²/R − mg
最低点：N₂ − G = mv₂²/R → N₂ = mv₂²/R + mg
压力差：ΔN = N₂ − N₁
```

**参数**：`m`, `R`, `g`, `v1`, `v2`

**Goals**：
```python
'force_top':    [r'最高点.*压力', r'最高点.*拉力', r'N₁', r'N1']
'force_bottom': [r'最低点.*压力', r'最低点.*拉力', r'N₂', r'N2']
'force_diff':   [r'压力差', r'ΔN']
```

**Bricks（5块）**：
```
intro     — 题目介绍（展示轨道+小球在最低点）
top       — 小球到最高点，显示受力
bottom    — 小球到最低点，显示受力
swing     — 运动一周（完整圆周动画）
sol_N1    — 求 N₁ [steps]
sol_N2    — 求 N₂ [steps]
sol_dN    — 求 ΔN [steps]
ending    — 答案汇总
```

**Layers**：`bg`, `env`(圆形轨道), `object`, `forces`, `data`, `title`, `panel`, `ending`

**Physics 函数**：
```js
function circularPos(prog) {
  var angle = prog * 2 * Math.PI;
  var px = CX + SC * Math.sin(angle);
  var py = CY - SC * Math.cos(angle);
  return { px, py, angle, /* 根据angle计算v,N1,N2 */ };
}
```

**特殊注意**：需要 canvas 绘制圆形轨道、小球在轨道上的位置、向心力和重力的箭头。

**总改动量**：替换 JS 部分（~250行）

**Bricks（5块）**：

### C2. `collision.html` — 动量守恒/碰撞

**物理模型**：
```
动量守恒：m₁v₁ + m₂v₂ = m₁v₁' + m₂v₂'
弹性碰撞：v₁' = (m₁−m₂)v₁/(m₁+m₂) + 2m₂v₂/(m₁+m₂)
         v₂' = 2m₁v₁/(m₁+m₂) + (m₂−m₁)v₂/(m₁+m₂)
完全非弹性：v' = (m₁v₁ + m₂v₂)/(m₁+m₂)
能量损失：ΔE = ½m₁v₁² + ½m₂v₂² − ½(m₁+m₂)v'²
```

**参数**：`m1`, `m2`, `v1`, `v2`, `elastic`

**Bricks（5块）**：
```
intro      — 两球初始位置
approach   — 接近过程
collide    — 碰撞瞬间（慢放）
depart     — 碰后分离
sol_vel    — 求碰后速度
sol_eloss  — 求能量损失
ending     — 答案汇总
```

**HTML 结构**：约 800 行新文件

## 公共架构

### 模板文件统一结构

每个模板 HTML 遵循以下结构（约 600-1000 行）：

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>题型 · 讲解动画</title>
  <style>/* 统一样式 + 题型特有样式 */</style>
</head>
<body>
  <div class="app">
    <canvas id="mainCanvas"></canvas>
    <div class="known-panel" id="knownPanel"></div>
    <div class="solution-panel" id="panel"></div>
    <div class="scene-overlay"><div class="scene-title" id="sceneTitle"></div></div>
    <div class="ending-bg" id="endingBg"><div class="ending-card"><h2>答案汇总</h2><div id="endingAnswers"></div></div></div>
    <div class="controls"><!-- playBtn, rewindBtn, progressBar, timeLabel, speedBtn --></div>
  </div>
  <script>
  (function(){
  'use strict';
  // 1. 参数 + 物理常量
  // 2. BRICKS + composeScenes + allocateTime
  // 3. 物理引擎函数
  // 4. FC (帧控制器) — 每个模板相同
  // 5. SD (场景调度) — 每个模板相同
  // 6. R (Canvas渲染器) — layers-based
  // 7. Panel (动态步骤面板) — 每个模板相同
  // 8. Ending (答案汇总) — 每个模板相同
  // 9. KnownPanel (已知量) — 每个模板相同
  // 10. UI (控制栏) — 每个模板相同
  // 11. onFrame + 启动
  })();
  </script>
</body>
</html>
```
