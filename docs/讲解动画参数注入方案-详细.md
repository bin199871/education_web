## 2. 注入方案

### 2.1 render() 注入方式

```python
def render(self, template_id: str, params: dict) -> str:
    tpl = self.registry.get(template_id)
    if not tpl:
        raise ValueError(f'未知模板: {template_id}')
    template_path = self.templates_dir / tpl['file']
    if not template_path.exists():
        raise FileNotFoundError(f'模板文件不存在: {template_path}')

    with open(template_path, 'r', encoding='utf-8') as f:
        html = f.read()

    # 在 <head> 后注入参数
    inject = json.dumps(params, ensure_ascii=False)
    script = f'<script>var __INJECTED_PARAMS__ = {inject};</script>\n'
    html = html.replace('<head>', '<head>\n' + script)

    # 标题替换
    title = params.get('title', tpl['label'])
    html = html.replace('<title>', f'<title>{title} · ')

    # 生成时间
    gen_time = datetime.now().strftime('%Y-%m-%d %H:%M:%S')
    html = html.replace('</body>', f'<!-- 生成时间: {gen_time} -->\n</body>')

    return html
```

```python
PARAM_MAP = {
    'electric_pendulum': {
        'mass':            'm',
        'charge':          'q',
        'electric_field':  'E',
        'pendulum_length': 'L',
        'g':               'g',

    },
    'projectile': {
        'initial_velocity': 'v0',
        'height':           'h',
        'g':                'g',
    },
    'vertical_circular': {
        'mass': 'm',
        'g':    'g',
    },
    'conveyor_belt': {
        'mass': 'm',
        'mu':   'mu',
        'g':    'g',
    },
    'board_block': {
        'mass': 'm',
        'mu':   'mu',
        'g':    'g',
    },
}


def map_params(template_id: str, extracted: dict) -> dict:
    """将 extract_physics_params() 输出的 params 映射为模板 schema 的 key。"""
    mapping = PARAM_MAP.get(template_id, {})
    result = {}
    for ext_key, schema_key in mapping.items():
        value = extracted.get(ext_key)
        if value is not None:
            result[schema_key] = value
    return result
```

## 4. API 改动

### 4.1 template_generate() 修改

```python
@app.post("/api/template/generate")
async def template_generate(req: TemplateGenRequest):
    text = req.text.strip() if req.text else ''
    if not text:
        raise HTTPException(status_code=400, detail='题目文本不能为空')

    engine = get_engine()
    tid = engine.match_template_by_text(text)
    if not tid:
        return {'success': False, 'fallback': True, 'message': '未匹配到动画模板'}

    # 提取参数并映射
    physics = extract_physics_params(text)
    mapped = map_params(tid, physics.get('params', {}))

    # 可选：将 template 默认值填入未提取到的参数
    tpl = engine.registry.get(tid)
    if tpl:
        for key, schema in tpl.get('param_schema', {}).items():
            if key not in mapped:
                mapped[key] = schema.get('default')

    try:
        html = engine.render(tid, mapped)
    except FileNotFoundError as e:
        raise HTTPException(status_code=500, detail=str(e))

    output_path = STATIC_DIR / 'premium-output.html'
    with open(str(output_path), 'w', encoding='utf-8') as f:
        f.write(html)

    return {
        'success': True,
        'template_id': tid,
        'html': html,
        'fallback': False,
        'message': '生成成功',
    }
```

## 6. 解题步骤答案的动态化

目前解题步骤 HTML 中的数值是硬编码的。理论上最优做法是全部动态化，但工作量较大。按优先级分为两阶段：

### 6.1 第一阶段（本次实现）

- 参数注入生效：用户输入的 m, q, E, L, g 等反映到物理模拟和画布渲染上
- 已知量面板动态显示
- **解题步骤文本中的数值保持静态不变**（仍显示默认值的计算结果）

### 6.2 第二阶段（后续扩展）

- 解题步骤中的数值改为 JS 模板字符串动态计算
- `PARAMS.questions` 数组中的 answer 改为动态计算
- 需要针对每个模板逐个改写步骤 HTML 和 `End` 组件

**以 inclined_plane 为例的第二阶段改造：**

## 7. 边界情况处理

### 7.1 部分参数未提取到

`extract_physics_params()` 可能只提取到部分参数。例如输入"一质量 2kg 的物体从 5m 自由落下"可能提取到 `mass=2, height=5`，但 g 未提取。此时：

1. `map_params()` 只映射提取到的 key
2. 在 `template_generate()` 中将 `param_schema` 中的 default 填入空缺
3. 模板 JS 中 `??` 回退到 `_DEFAULTS`

回退链：`__INJECTED_PARAMS__` → `_DEFAULTS`（硬编码默认值）

### 7.2 参数类型不一致

`extract_all_params()` 输出的值都是 float，模板 schema 也接受 float。无需额外类型转换。

### 7.3 未知的模板 ID

`map_params()` 对未知 template_id 返回空字典，render 仍能正常渲染（使用模板自身默认值）。

### 7.4 %s 和 SCENES 语法错误

`conveyor_belt.html` 和 `board_block.html` 中的 `SCENES` 数组使用了无效的裸变量 `{id: s, s: 0, e: 0.1}`。本次注入改造**不修复**渲染逻辑问题，只注入参数。渲染错误的修复留待后续。
