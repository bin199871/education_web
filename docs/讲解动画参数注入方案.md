# 讲解动画参数注入方案

## 问题

当前【讲解动画】按钮的完整流程：

```
题目文本 → match_template_by_text() → render(tid, {}) → 返回 HTML
```

`render()` 试图做三个替换，但全部无效：

| 替换目标 | 模板中是否存在 | 结果 |
|---|---|---|
| `var TEMPLATE_PARAMS = null;` → 参数 JSON | ❌ 6 个模板文件都没有 | 空操作 |
| `[[TITLE]]` → 标题 | ❌ 6 个模板文件都没有 | 空操作 |
| `[[GENERATED_TIME]]` → 时间 | ❌ 6 个模板文件都没有 | 空操作 |

模板的物理参数全部**硬编码**在 JS 中。例如 `electric_pendulum.html`：

```js
const PHYS = {
  m:.1, q:5e-4, E:2000, g:10, L:1,
  ...
}
```

因此用户输入的题目参数未被提取，也未注入模板。

---

## 方案：四步

### 第一步：统一注入接口

修改 `TemplateEngine.render()`，不再依赖模板中不存在的占位符字符串，而是在 `<head>` 后注入一段全局 `<script>`。

```python
def render(self, template_id: str, params: dict) -> str:
    tpl = self.registry.get(template_id)
    if not tpl:
        raise ValueError(f'未知模板: {template_id}')
    template_path = self.templates_dir / tpl['file']
    if not template_path.exists():
        raise FileNotFoundError(f'模板文件不存在: {template_path}')
    with open(template_path, 'r', encoding='utf-8') as f:
        html = f.read()
    # 在所有模板 head 后注入参数
    inject_script = (
        '<script>'
        f'var __INJECTED_PARAMS__ = {json.dumps(params, ensure_ascii=False)};'
        '</script>\n'
    )
    html = html.replace('<head>', '<head>\n' + inject_script)
    # 渲染标题
    title = params.get('title', tpl['label'])
    html = html.replace('<title>', f'<title>{title} · ')
    gen_time = datetime.now().strftime('%Y-%m-%d %H:%M:%S')
    html = html.replace('</body>', f'<!-- 生成时间: {gen_time} -->\n</body>')
    return html
```

这样每个模板在运行时通过 `__INJECTED_PARAMS__` 拿到外部参数，不存在则为 `undefined`。

### 第二步：模板 JS 适配

修改每个模板文件，在初始化时检查 `__INJECTED_PARAMS__`，有则覆盖默认值。

**6 个模板都需要改动：**

| 模板 | 硬编码位置 |
|---|---|
| `electric_pendulum.html` | `const PHYS = { m:.1, q:5e-4, ... }` |
| `inclined_plane.html` | 类似 `const PHYS` 结构 |
| `projectile.html` | 同上 |
| `vertical_circular.html` | 同上 |
| `conveyor_belt.html` | 同上 |
| `board_block.html` | 同上 |

**修改模式**（以 `electric_pendulum.html` 为例）：

```js
// 原本：
const PHYS = {
  m:.1, q:5e-4, E:2000, g:10, L:1,
  get Fq(){return this.q*this.E},
  get G(){return this.m*this.g},
  ...
}

// 改为：
const _DEFAULTS = {
  m:.1, q:5e-4, E:2000, g:10, L:1
};
const _IN = (typeof __INJECTED_PARAMS__ !== 'undefined') ? __INJECTED_PARAMS__ : {};
const PHYS = {
  m: _IN.m ?? _DEFAULTS.m,
  q: _IN.q ?? _DEFAULTS.q,
  E: _IN.E ?? _DEFAULTS.E,
  g: _IN.g ?? _DEFAULTS.g,
  L: _IN.L ?? _DEFAULTS.L,
  get Fq(){return this.q*this.E},
  get G(){return this.m*this.g},
  ...
};
```

### 第三步：参数映射

`extract_physics_params()` 提取的 key 与模板 `param_schema` 的 key 命名不一致，需要映射。

```python
# 每个模板关注的 key 映射表
PARAM_MAP = {
    'electric_pendulum': {
        'mass':            'm',
        'charge':          'q',
        'electric_field':  'E',
        'pendulum_length': 'L',
        'g':               'g',
    },
    'inclined_plane': {
        'mass':         'm',
        'angle_deg':    'theta',
        'slope_length': 'L',
        'mu':           'mu',
        'g':            'g',
    },
    'projectile': {
        'initial_velocity': 'v0',
        'height':           'h',
        'g':                'g',
    },
    'vertical_circular': {
        'mass':  'm',
        'g':     'g',
    },
    'conveyor_belt': {
        'mass': 'm',
        'mu':   'mu',
        'g':    'g',
    },
    'board_block': {
        'mass': 'm',
        'mu':   'mu',
        'g':    'g',
    },
}


def map_params(template_id: str, extracted: dict) -> dict:
    """将 extractor 输出的 key 映射为模板 schema 的 key。"""
    mapping = PARAM_MAP.get(template_id, {})
    result = {}
    for ext_key, schema_key in mapping.items():
        if ext_key in extracted and extracted[ext_key] is not None:
            result[schema_key] = extracted[ext_key]
    return result
```

### 第四步：整合到 API

修改 `server.py` 中的 `template_generate()`：

```python
@app.post("/api/template/generate")
async def template_generate(req: TemplateGenRequest):
    text = req.text.strip() if req.text else ''
    if not text:
        raise HTTPException(status_code=400, detail='题目文本不能为空')
```

## 关于动态计算

参数注入后，部分物理量（如摆角 θ₀、电场力 Fₑ、平衡位置等）在模板内由 JS 基于 `PHYS` 实时计算。例如 `electric_pendulum.html` 中的 `swing1()`、`swing2()` 函数基于 `PHYS` 做数值模拟。这种情况下只需要正确注入原材料参数（m, q, E, L, g），衍生量由模板 JS 自行推导即可。

但对于某些模板（如 `inclined_plane`），如果场景进度、帧数等与参数强相关，可能需要进一步修改 JS 计算逻辑，不局限于简单的 ?? 回退。
