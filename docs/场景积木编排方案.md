# 场景积木编排方案 — 详细设计

## 问题本质

当前 electric_pendulum 模板被绑定到一个**特定的题目**：

```
从 0° 释放 → 求 θ=37° 时的速度 → 拉到 90° 释放 → 求最大拉力
```

用它去播结构不同的题目（如 60° 释放 → 求 vA → 求 ΔEp → 求 Tmax），即使参数对了，**场景叙事也不匹配**。

## 核心思路：积木式组合

把模板拆成**场景积木（Scene Bricks）**，每块积木解决一个**子问题**。根据题目实际问什么，动态选择和拼接积木。

## 设计决策（已确认）

| # | 问题 | 决定 |
|---|---|---|
| 1 | 积木粒度 | **细粒度** — 摆过程与解答分开 |
| 2 | 过渡场景 | **保留**，每个 sol 场景自带 `transIn` |
| 3 | highlight | **保留**，每个 sol 场景自带 `highlight` |
| 4 | 跨模板通用性 | **设计时考虑**，通用积木放在 `_COMMON` 中 |

## 3. 编排引擎

### 规则

```
composeScenes(goals) → [
  intro ── 通用，总是存在
  forces ── 通用，总是存在
  for each goal in goals:
    匹配积木 → 按顺序插入
  ending ── 通用，总是存在
]

每个 sol 积木自动获得:
  transIn: 显示过渡文字，占前 15% 时间
  highlight: 慢速播放，占前 60% 时间
  steps: 逐条显示，占 85% 时间后弹出答案
```

### 编排伪代码

```js
function composeScenes(goals) {
  const has = (g) => goals.includes(g);
  const list = [];

  list.push('intro', 'forces');

  if (has('force_analysis') || has('resultant_force'))
    list.push('sol_force');

  if (has('velocity_bottom'))
    list.push('swing_to_bottom', 'sol_bottom_velocity');
  else if (has('velocity'))
    list.push('swing_to_angle', 'sol_velocity_at_angle');

  if (has('energy_change'))
    list.push('sol_energy');

  if (has('tension_max'))
    list.push('swing_to_bottom_2', 'sol_tension');

  list.push('ending');
  return allocateTime(list);
}
```

## 5. 渲染器

改为 `layers` 声明式渲染：

```js
render(sceneId, p) {
  const scene = SCENES.find(s => s.id === sceneId);
  if (!scene) return;
  const layers = scene.layers || ['bg'];
  ctx.clearRect(0, 0, CV.W, CV.H);
  if (layers.includes('bg'))       this.drawBg();
  if (layers.includes('efield'))   this.drawEField(p.by);
  if (layers.includes('env'))      this.drawEnv();
  if (layers.includes('pend'))     this.drawPend(p.bx, p.by, p.theta);
  if (layers.includes('forces'))   this.drawForcesFull(p.bx, p.by);
  if (layers.includes('v_arrow'))  this.drawVArrow(p.bx, p.by, p.theta, p.v);
  if (layers.includes('energy'))   this.drawEnergy(p);
  if (layers.includes('angle'))    this.drawAngle(p.theta);
  if (layers.includes('tension'))  this.drawTension(p.bx, p.by, p.T);
  if (layers.includes('title'))    this.drawTitle(scene.title);
  if (layers.includes('known'))    this.drawKnown();
}
```

新增 `drawTension` 函数（当前 `drawForcesSimp` 中已有张力箭头逻辑，将其提取为独立函数）。

## 6. 步骤面板

从静态 HTML 改为 JS 动态生成：

```html
<!-- 面板容器不变，内容由 JS 填充 -->
<div class="solution-panel" id="panel">
  <div id="panel-dynamic-content"></div>
</div>
```

```js
Panel.renderSteps(steps) {
  const container = document.getElementById('panel-dynamic-content');
  if (!container) return;

  let html = `<div class="panel-title">${steps.title}</div>`;
  steps.items.forEach((item, i) => {
    html += `<div class="step-row" id="dyn_step_${i}">`;
    html += `<span class="step-badge">${i+1}</span>`;
    html += `<span>${item}</span></div>`;
  });
  html += `<div class="answer-box" id="dyn_answer"><span class="answer-text">${steps.answer}</span></div>`;
  container.innerHTML = html;
}
```
