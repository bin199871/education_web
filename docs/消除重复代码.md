# 消除重复代码 · 改造说明

> 本文档记录 `template_builder.py` 的产生原因、改造方案、使用方法
> 以及出现问题时如何回退。

---

## 1. 为什么要消除重复代码

### 1.1 现状

16 个物理动画模板，每个都是一个独立的 HTML 文件，其中有大量重复代码：

| 模块 | 行数 | ×16个模板 | 总重复量 |
|------|------|-----------|---------|
| 帧控制器 FC | 25行 | 16 | 400行 |
| 场景调度 SD | 10行 | 16 | 160行 |
| 解题面板 Panel | 25行 | 16 | 400行 |
| 场景标题 ST | 8行 | 16 | 128行 |
| 答案汇总 End | 12行 | 16 | 192行 |
| 已知量 KP | 12行 | 16 | 192行 |
| UI 控制器 | 20行 | 16 | 320行 |
| onFrame + 启动 | 14行 | 16 | 224行 |
| 星星生成 | 6行 | 16 | 96行 |
| CSS 样式 | 200行 | 16 | 3200行 |
| **合计** | **~332行** | **16** | **~5312行重复** |

### 1.2 重复代码带来的问题

1. **修 bug 要改 16 个文件** —— 之前 `_lastT=dt` 的 bug 需要逐一修改所有模板
2. **新增模板工作量大** —— 每个新模板要复制粘贴 332 行公共代码
3. **代码不一致** —— 16 个模板的同一份逻辑可能出现细微差异
4. **代码审查困难** —— 分不清改的是"业务逻辑"还是"基础设施"

## 2. 改造方案

### 2.1 架构变化

```
改造前:                         改造后:
                                  template_common.js  (公共JS)
                                  template_common.css (公共CSS)
┌─────────────────┐               ┌───────────────────────┐
│ template_1.html  │  (全量)       │ template_parts/1.js    │ (仅特有代码)
│ template_2.html  │  (全量)       │ template_parts/2.js    │ (仅特有代码)
│ ...              │               │ ...                     │
│ template_16.html │  (全量)       │ template_parts/16.js   │ (仅特有代码)
└─────────────────┘               │ template_builder.py    │  (组装脚本)
                                  │ templates/1.html       │ (生成的完整文件)
                                  │ templates/...          │
                                  └───────────────────────┘
```

### 2.2 文件职责

| 文件 | 内容 | 维护者 |
|------|------|--------|
| `template_common.js` | 帧控制器、场景调度、面板、UI控制器等 **不变量** | 核心开发 |
| `template_common.css` | 全局样式（星空、面板、控制栏等） | 核心开发 |
| `template_parts/<id>.js` | 每个模板独有的参数、场景、物理引擎、渲染器 | 题型开发者 |
| `template_builder.py` | 读取 parts + common → 组装为完整 HTML | 核心开发 |
| `templates/<id>.html` | 生成的完整 HTML（由 builder 自动生成） | **永不手动修改** |

### 2.3 部件文件格式

每个 `template_parts/<id>.js` 通过标记段（section marker）定义模板的特有内容：

```javascript
// BGN_CSS
/* 该模板独有的CSS（如果有） */
.color-m1{color:#60a5fa}
// END_CSS

// BGN_KNOWN_ROWS
<div class="known-row" id="kmM">m = 0.5 kg</div>
<div class="known-row" id="kmR">R = 1.0 m</div>
// END_KNOWN_ROWS

// BGN_PANEL_HTML
<div class="panel-content" id="panel-q1" style="display:none">
  <div class="panel-title" style="color:#93c5fd">最高点：求支持力 N₁</div>
  <div class="step-row" id="s1s0"><span class="step-badge">①</span><span>最高点受力</span></div>
  <div class="answer-box" id="s1ans"><span class="answer-text">答案</span></div>
</div>
// END_PANEL_HTML

// BGN_PARAMS_DEFAULTS
{"m": 0.5, "R": 1.0, "g": 10, "v1": 5, "v2": 8,
 "title": "竖直圆周运动",
 "questions": [{"title":"N₁","answer":"N₁=?"},{"title":"N₂","answer":"N₂=?"},{"title":"ΔN","answer":"ΔN=?"}]}
// END_PARAMS_DEFAULTS

// BGN_SPECIFIC_JS
// 物理常量
const P = Object.assign({m:0.5,R:1.0,g:10,v1:5,v2:8}, PARAMS);

// 预计算结果
const N1 = P.m*P.v1*P.v1/P.R - P.m*P.g;
const N2 = P.m*P.v2*P.v2/P.R + P.m*P.g;
const dN = N2 - N1;

// 画布常量
const CX=480, CY=320, SCALE=250, BR=14;
const TOTAL=9000, FPS=60;

// 场景定义
const SCENES = [
  {id:'intro',s:0.00,e:.08},{id:'force_top',s:.08,e:.18},{id:'force_bot',s:.18,e:.28},
  {id:'sol_q1',s:.28,e:.38},{id:'trans1',s:.38,e:.42},{id:'swing',s:.42,e:.55},
  {id:'highlight',s:.55,e:.60},{id:'sol_q2',s:.60,e:.70},{id:'sol_q3',s:.70,e:.82},{id:'ending',s:.82,e:1}
];
SCENES.forEach(s=>{s.sf=Math.floor(s.s*TOTAL);s.ef=Math.floor(s.e*TOTAL);s.dur=s.ef-s.sf});

// 场景标题映射
ST._titles = {
  intro:'竖直圆周运动',force_top:'最高点受力分析',force_bot:'最低点受力分析',
  swing:'小球沿轨道运动一周',highlight:'最高点与最低点对比',
  sol_q1:'求N₁',sol_q2:'求N₂',sol_q3:'求ΔN'
};

// 解题面板注册
Panel.panels = {q1:document.getElementById('panel-q1'),q2:document.getElementById('panel-q2'),q3:document.getElementById('panel-q3')};
Panel.stepMap = {sol_q1:{pf:'s1',cnt:5,ans:'s1ans'},sol_q2:{pf:'s2',cnt:4,ans:'s2ans'},sol_q3:{pf:'s3',cnt:4,ans:'s3ans'}};

// 已知量面板
KP._initFn = function(){
  function st(id,v,u){var e=document.getElementById(id);if(e)e.textContent=e.textContent.split('=')[0]+'= '+v+(u?' '+u:'');}
  st('kmM',P.m,'kg');st('kmR',P.R,'m');st('kmV1',P.v1,'m/s');st('kmV2',P.v2,'m/s');st('kmG',P.g,'m/s²');
};

// 物理引擎
function GET_PHYS(sceneId,prog){
  function getCirc(theta){
    const px=CX+SCALE*Math.sin(theta), py=CY-SCALE*Math.cos(theta);
    const h=P.R*(1-Math.cos(theta));
    const v=Math.sqrt(Math.max(0,P.v2*P.v2-2*P.g*h));
    const Fc=P.m*v*v/P.R;
    const N=Fc+P.m*P.g*Math.cos(theta);
    return {theta,px,py,v,Fc,N,h,deg:theta*180/Math.PI};
  }
  switch(sceneId){
    case 'intro': return getCirc(0);
    case 'force_top': return getCirc(Math.PI);
    case 'sol_q1': return getCirc(Math.PI);
    case 'force_bot': return getCirc(0);
    case 'swing': return getCirc(prog*Math.PI*2);
    case 'highlight': return getCirc(prog*Math.PI*2);
    case 'sol_q2': case 'sol_q3': return getCirc(0);
    default: return getCirc(0);
  }
}

// 渲染器
const RENDER = {
  render: function(sceneId,ph){
    var ctx=this.ctx,px=ph.px,py=ph.py;
    ctx.clearRect(0,0,960,640);
    this.drawBg();
    this.drawTrack();
    this.drawBall(px,py);
    // ... 具体绘制代码
  },
  // ... 辅助方法
};
// 更新答案的函数
// 函数名必须为 updateAnswers，builder 会在 html 中自动调用
function updateAnswers(){
  document.querySelector('#s1ans .answer-text').textContent = 'N₁ = '+N1.toFixed(2)+' N';
  document.querySelector('#s2ans .answer-text').textContent = 'N₂ = '+N2.toFixed(2)+' N';
  document.querySelector('#s3ans .answer-text').textContent = 'ΔN = '+dN.toFixed(2)+' N';
}
// END_SPECIFIC_JS
```

## 3. 如何使用 builder

### 3.1 首次使用

```bash
# 1. 生成公共文件（template_common.js + template_common.css）
cd backend
python template_builder.py init

# 2. 构建所有已有模板的部件文件
# （这一步需要为每个模板创建对应的 template_parts/<id>.js）
# （参见第6章"回退方案"→建议：保留原始文件不动）

# 3. 构建单个模板
python template_builder.py build vertical_circular

# 4. 构建全部
python template_builder.py build_all
```

### 3.2 日常开发流程

```bash
# 创建新模板时：
1. 在 template_parts/ 下创建 <新题型>.js
2. 编写特有代码（场景、物理、渲染）
3. 运行 build <新题型> 生成完整 HTML
4. 测试生成的 templates/<新题型>.html

# 修改公共代码时：
1. 修改 template_builder.py 中的 COMMON_JS_CONTENT 或 COMMON_CSS_CONTENT
2. 运行 build_all 重新生成所有模板
3. 验证生成的文件

# 修改某个模板的特有逻辑时：
1. 编辑 template_parts/<id>.js
2. 运行 build <id> 重新生成
```

### 3.3 builder 命令参考

| 命令 | 作用 |
|------|------|
| `python template_builder.py init` | 生成公共 JS/CSS 文件 |
| `python template_builder.py list` | 列出所有部件文件 |
| `python template_builder.py build <id>` | 构建指定模板 |
| `python template_builder.py build_all` | 构建所有模板 |

## 5. 风险与注意事项

| 风险 | 影响 | 应对 |
|------|------|------|
| 公共 JS 修改后影响所有模板 | 高 | 修改后必须 build_all + 逐个测试 |
| 部件文件语法错误 | 仅该模板无法构建 | build 时检查 stdout 错误信息 |
| 部件文件中的标记段遗漏 | 构建出的 HTML 缺内容 | 检查日志中的 section 提取结果 |
| template_engine.py 和 builder 不同步 | 注册了模板但没部件文件 | `list` 命令对比注册表和部件列表 |
| 公共 JS 中使用了未定义的变量 | 所有模板加载时报错 | 公共 JS 只使用 `TOTAL`/`FPS`/`SCENES`/`PARAMS`/`GET_PHYS`/`RENDER` 六个外部变量 |

### 5.1 公共 JS 依赖的外部变量

公共 JS 中引用了以下变量名，它们必须在部件 JS 中定义：

| 变量 | 类型 | 定义位置 | 用途 |
|------|------|---------|------|
| `TOTAL` | int | 部件 JS | 总帧数 |
| `FPS` | int | 部件 JS | 帧率（通常 60） |
| `SCENES` | array | 部件 JS | 场景定义 |
| `PARAMS` | object | 部件 JS / builder | 参数 |
| `GET_PHYS` | function | 部件 JS | 物理状态获取 |
| `RENDER` | object | 部件 JS | 渲染器 |

## 6. 回退方案（出现问题时使用）

### 6.1 方法一：使用 Git 回退（推荐）

如果问题在改造过程中出现，且代码已提交到 Git：

```bash
# 查看改动了哪些文件
git status

# 回退到上一个稳定版本
git checkout -- backend/templates/       # 恢复所有模板到 git 版本
git checkout -- backend/template_*.py    # 恢复 builder 和 common 文件
git checkout -- frontend/physics-*.html  # 恢复前端文件

# 如果需要完全回退到某个 commit
git log --oneline -10                    # 查看最近 10 个 commit
git reset --hard <commit-hash>           # 回退到指定 commit
```

### 6.2 方法二：保留原始文件不动（预防措施）

**builder 改造不会删除或修改原始文件。** 原始模板文件仍保留在：

```
backend/templates/electric_pendulum.html   ← 原始文件（手动修改前的版本）
backend/templates/inclined_plane.html      ← 原始文件
...
```

builder 生成的同名文件会覆盖 `backend/templates/` 下的文件，
但 `frontend/` 目录下的文件不受影响（如果预览用的是 `frontend/` 的副本）。

**安全做法：在 builder 初次运行前备份原始模板：**

```bash
cp -r backend/templates backend/templates.bak.$(date +%Y%m%d)
```

回退时只需：

```bash
cp -r backend/templates.bak.20260627/* backend/templates/
```

### 6.3 方法三：直接禁用 builder，回到旧模式

如果 builder 引入的问题难以修复，可以完全禁用 builder：

1. **停止使用 builder** —— 改为直接编辑 `backend/templates/<id>.html`
2. **忽略 `template_parts/` 目录** —— 所有修改直接在生成的完整 HTML 上进行
3. **删除公共文件** —— `template_common.js` 和 `template_common.css` 可以删除而不影响任何功能

这样每个模板又恢复到独立的完整性，代价是后续修改公共代码需要逐文件操作。

## 7. 改造后的问题排查

### 7.1 构建报错

```bash
python template_builder.py build electric_pendulum
# 如果报错，检查：
# 1. template_parts/electric_pendulum.js 是否存在
# 2. 文件中的标记段 // BGN_XXX 和 // END_XXX 是否完整配对
# 3. JS 语法是否正确（可以用 node -e "检查"
```

### 7.2 生成的模板无法播放

```bash
# 1. 检查浏览器控制台错误信息
# 2. 确认公共 JS 中引用的变量在部件 JS 中已定义
# 3. 对比改造前后生成的 HTML 差异
diff -u <(git show HEAD:templates/electric_pendulum.html) templates/electric_pendulum.html
```

### 7.3 所有模板同时出现问题

```bash
# 大概率是公共 JS 改动导致
# 1. 检查 template_builder.py 中的 COMMON_JS_CONTENT
# 2. 回退到上一次有效的版本
git checkout -- backend/template_common.js backend/template_common.css
python template_builder.py build_all
```
